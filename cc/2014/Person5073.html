Summary

Spyros Doulgeridis is holding one Master degree in Telecommunications and one in Software Engineering. He has a huge experience in JavaEE development especially backend implementations; and also has 3 years of experience in Oracle Application Development Framework (ADF) in major ADF project, one fo Oracle success stories, specializing in all layer of the application from front-end development to database programming and from designing ADF forms to Continuous Integration Solutions.

He is a trustworthy professional who is used to work under strict deadlines in very demanding and complex projects that require high quality code. He is well known for his friendly personality and soft skills. His wide knowledge gives him the ability to comprehent customer problems in every aspect providing accurate solutions. He faces difficulties as challenges that help to both improve personality and build character.

Oracle ADF Consultant/Software Engineer. During the last year, I was fortunate to participate in one of the first attempts to migrate an old big scale Oracle Forms 6i application (around 250 forms) to Oracle ADF 11g (more on Oracle Success stories).
Reengineering the old application in order for the new application and the old one to work in parallel imposes extra difficulties as it was necessary to respect the approach of the old app. During the project I was involved both in developing (in my first months), design reviewing and developer’s assisting:
• Developing using all the features that the ADF framework provides such as: ADF Business Components, ADF Task Flows, ADF Faces Rich Client (JSF, ADF rich AJAX components, Trinidad components etc). Also I was partially involved in the deploying of the application on clustered Weblogic 11g (10.3.1) server.
• Designing: Although ADF is considered to be maybe the best choice for migrating a Forms application (Client-Server) into a web based one, unfortunately there are some incompatibilities and problems when trying to implement the same functionality in Forms. During the project I was also involved in providing the proper design based on the analysis of the functionality of the existing Oracle Forms 6i application, so as to provide solutions in order for the migration to be feasible.
• Reviewing: Code reviewing, inspecting the implementation and functionality of the work performed by developers, checking the developing decisions that were adopted, verifying that Company’s Developing Standards were followed in order for the deliverable to reach the expected quality state before Testing.
• Developers’ assisting: Helping developers to workaround problems, implement the functionality by utilizing the framework features and avoiding discourage practices.
