Luke Kenneth Casson Leighton’s Summary:

My vocation is to seek out, understand, document and then explain all the archaic, esoteric and obscure technologies i come into contact with. So far, that has led me to network-reverse-engineering of Windows NT 4.0 Domains; reverse-engineering the High Tech Corporation's Wince Mobiles in order to run Linux on them just for fun; Adopting Python as a core programming skill before it became fashionable; Writing an XML-based programming language before _that_ became fashionable.

The upshot of all this is that there really isn't very much in Computing Technology that I would particularly find difficult - or if it _is_ difficult, I'll be able to tell you why; and if I've not encountered a technology before, I'll tell you so, up-front, and then go find out about it for you.

When doing programming, I much prefer to work with open source code, simply because any company that sells proprietary systems leads its clients into vendor lock-in (but if there genuinely isn't anything else out there, count me in if it's cool technology you want to develop!). Free software Development is much more honest: you're getting a service. Don't like the service? sack my ass and get someone else: you STILL keep the code I worked on (and so do I).
