  • Senior Applications Engineer

    Digital Railroad

    (Privately Held; Photography industry)

    April 2006 — October 2008 (2 years 7 months)

    Lead a team of 4+ developers and web developers in an effort to become the first player in the industry to fully support PLUS (Photo Licensing Universal Standard: useplus.org) for rights managed photography. I designed the entire back-end and coded a significant portion using MS SQL Server, C#.NET and some Ruby on Rails.

    I revamped the company’s event reporting system. Event ‘ingestion’ Windows Services on event generating servers moved data from xml log files to intermediate SQL databases. Event ‘digestion’ Window Services on another machine moved the ‘ingested’ SQL event data into the user-accessed event SQL database. This two-step design decreased event visibility latency from 4+ hours to minutes while maintaining minimal load on web and other production servers.

    Lead a team to transition the site’s entire ecommerce system from one using only USD to one where users can price and view images in their choice currency. Overcome issues include maintaining up-to-date exchange rate data in the application and ensuring that no funds are lost/gained between buyers and sellers in cross-currency transactions.

    Solely conceived, designed and developed the development team’s system of syncing and source control for SQL objects in our development environments. With this system, the change history of stored procedures, tables, views, constraints, indexes, etc. is tracked per object in the same way one would use source control with class and project files. The system is written in C#.NET and utilizes Red Gate.

    Designed and implemented many of the algorithms used to ‘rescue’ and re-associated orphaned images and metadata with the proper owners when our main database server and backups for two months were lost. No, it wasn’t fun, and no, I didn’t sleep. I first wrote a file crawler to discover the orphan images and then referenced a logging database to deduce the images’ proper owners. Over 99.9% of the over 650,000 orphaned images were recovered with 100% accuracy.

  • Technology Associate

    Bridgewater Associates

    (Privately Held; Investment Management industry)

    July 2003 — April 2006 (2 years 10 months)

    Introduced performance monitoring, data harvesting, and data and process transparency to our process for the generation of individualized exposure reports for our clients. Also achieved towards the goal of developing an end-to-end reporting process was the ability to automatically build client reports according to the client’s schedule as well as on-the-fly. Technologies utilized by the process include web-pages (ASP.NET), web-services and windows applications (C#.NET), .NET Com-Interop and Reflection (C#.NET / VB 6 / Excel VBA), workflow tools (AgilePoint Envision) as well as a testing solution I developed that functionally tests the output of any application that creates office documents.

    Coordinated with Marketing, Client Service, and Operations to design and develop the company’s assets under management calculator. Using daily precalculated data, the calculator quickly pulls historical and current snapshots or timer-series of assets held, gained, or lost, broken down by client or account. The calculator is used for RFPs, external memos, as well as internal processes such as growth forecasting. Work completed using primary Microsoft SQL Server.

    For over one year, I was the sole programmer developing our prospective-client relationship management system that tracks prospects, competitors, clients, portfolios, contacts, mailing groups, events, follow-up items as well as relationship status (hits, leads, wins, losses, etc.). This included designing and developing a process to providing salespersons and client advisors with the ability to take a portion of the internal client-web offline on their laptops for use when internet connection is unavailable. Data is stored in SQL Server and xml. Applications and websites are written in ASP, ALP, ASP.NET, and C#.NET.
