  • Architect - Relational Engine

    Microsoft/SQL Server

    (Public Company; 10,001 or more employees; MSFT; Computer Software industry)

    January 2003 — June 2005 (2 years 6 months)

    Architect and development lead for object-relational extensions to the SQL Server 2005 relational engine to support a next-generation file system for Windows. The extensions include native persistence of objects defined by CLR classes. This includes support for object instances of sizes up to 2GB, type inheritance, complex embedded properties, and a record format to store object fragments as records in the database. SQL query and DML extensions to enable queries over inheritance, fine-grained updates to properties at any level of nesting in the structure of the object, and query optimizer extensions (indexing, statistics, and costing) to support efficient execution of these queries. In addition to the design and architecture I coordinated a virtual team of 20 software engineers and 3 program managers in the SQL relational engine.

  • Architect and developer

    Microsoft/SQL Server

    (Public Company; 10,001 or more employees; MSFT; Computer Software industry)

    January 2000 — December 2004 (5 years )

    Lead architect for the integration of the .NET CLR inside the SQL Server engine. This feature, shipped in SQL Server 2005, enables database application developers to write business logic and database extensions in the form of scalar and table-valued functions, procedures, triggers, types and aggregates using any .NET language. Designed and implemented large portions of the subsystem to support BLOBs and CLOBs in SQL 2005.
