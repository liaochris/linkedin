    LTE mobile user equipments emulator definition and developing in C/C++ for the NSN LTE eNodeB test.   • UMTS NodeB Call processing

    Nokia Siemens Networks

    (Public Company; NOK; Telecommunications industry)

    October 2007 — April 2008 (7 months)

    worked on the call processing software of the Siemens UMTS NodeB Base Band card (CHC96 project). Call processing is related to all those procedures needed for establishing connections with mobile equipments and configure and manage them. Activity was related to the maintenance of the software correcting it basing on modification requests come from test phase or from customers and the development of new features in particular the one related to an internal procedure for improving the resources allocation inside the board.
    The programming language used was C++ in a state machine and message based software in a real-time embedded environment, using real-time VxWorks OS and the ACE open source framework for concurrent communication software. The interface with the external NodeB network entities was CORBA.

  • sRIO Tracer tool

    Nokia Siemens Networks

    (Public Company; NOK; Telecommunications industry)

    May 2006 — September 2007 (1 year 5 months)

    design and development of a tool for tracing the communication between devices present in equipments for telecommunication. This communication is performed using a high-performance protocol called RapidIO. The tool was constituted by an AMC board with a Freescale PowerPC 8548 processor running a Linux OS (2.6.16). The application software was developed in C++ and controlled remotely by a PC via a web interface written in PHP. The tool allows the definition of triggering and filtering rules on the fields of messages exchanged between the software modules running on the equipment under analysis. These rules allow the user to define the events for starting or stopping the traffic capture and for filtering only useful information. Since the high-performance requirements of the RapidIO communication, a specific RapidIO driver for Linux has been developed for achieving these requirements for instance avoiding memory copies as much as possible moving only memory pointers.

  • WiMAX Base Station prototype

    Siemens

    (Telecommunications industry)

    May 2005 — April 2006 (1 year )

    worked on a WiMAX (802.16d/e) Base Station (BS) prototype. The goal was to create a testbed on which developing some WiMAX BS functionalities to verify/evaluate the performances needed in terms of processing resources as support to the pre-analysis task of the Siemens WiMAX BS. The hw platform used was a Freescale board mounting a PowerQuicc III 8560 processor. The software was developed in C using Freescale CodeWarrior IDE for all the software developing phases (code writing, compiling, debugging, etc). I was mainly involved on the development of L2 protocols and scheduler algorithms. The testbed environment was also constituted by a Mobile Stations simulator developed in C and running on a PC Linux, traffic generator and real application (FTP, VoIP, video streaming) for data traffic injection and various tools/scripts for the configuration and monitoring of the BS and the test environment
