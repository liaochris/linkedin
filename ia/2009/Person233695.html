Tim Johnson’s Summary

Software development for Linux, Windows, and various embedded platforms. Original Linux drivers include custom PCI hardware, network monitoring, and bio-engineering devices. Some web experience. Joint patent awarded for network device. OS kernel development. “Bare machine” software for USB, wireless, ATM and IP, Functional hardware design experience. A diverse application development background. Recent focus has been network applications, especially those related to network security.

Experience with Telecommunications, SSL Proxy, iptables, ICAP servers, IP routing/bridging, VoIP, ATM, TCP/IP. Protocol parsing experience including instant messenger protocols (Yahoo, AOL, ICQ) and graphic formats (JPEG, PNG, and GIF). Experience in fault tolerance/high availability, compiler tool chain development (compilers, assemblers and linkers), and SMP multi-processor kernel development.

Experience in the design and implementation of “pluggable” development environments: allowing work on a desktop environment to be transferred to an embedded environment without source code changes. A deep and broad understanding of the issues associated with multi-threaded and multi-processor environments.

    (Public Company; 51-200 employees; Computer Networking industry)

    February 2000 — December 2001 (1 year 11 months)

    Architect for the Anatel next generation VOIP gateway product called the TAP 900. The core to that product was a custom FPGA/ASIC called the TXB. The TAP 900 architecture was a blade product with an on-board processor (or local host), a DSP farm, and TXB switch interconnect between the local host processor, the DSP processors, and the network(s). Because the TXB provided both DSP to/from host and DSP to/from DSP interconnect, the architecture was suitable for conferencing, especially audio conferencing. Originally planned for Ethernet network connectivity, the same architecture was adapted for ATM UTOPIA Level 2 and AAL2. A PSTN interface to the board was also part of the architecture, but not necessarily required for all boards. Ethernet and IP routing related Linux driver changes.
