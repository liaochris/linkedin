  • Software Quality Engineer (contract)

    Getty Images

    (Privately Held; Photography industry)

    July 2001 — November 2002 (1 year 5 months)

    At the beginning of this assignment, it became apparent that there was quite a bit of overhead in the use of Excel for Test Case documentation. As repetitive and costly manual steps were identified, an Excel macro was developed which automated much of the effort. This automation was quickly adopted by the QA department, and used for several years.

    During my time at Getty Images, my SQL skills were developed. Initially, simple SQL queries were used to identify existing data that fit specific scenarios with which to exercise applications under test. Useful scripts that were saved and shared were developed from this practice, given testers the ability to quickly find existing test scenarios.

    There was an effort to develop an in-house Test Automation tool using VB. I was asked to be an early adopter of the tool by taking a few existing test cases and expanding them for other scenarios. The test scenarios were defined in Excel spreadsheets that were well over a hundred columns wide, representing each UI control to be manipulated, and specific data with which to manipulate them. It became immediately evident that this method of test case definition was infeasible, if not completely impossible to maintain.

    I made the suggestion at that time, based on my experience at Loudeye, to break down the scenario data into separate entities, then allow the user to combine various Entity-based scenarios to build a Test Case scenario. I further suggested that a "Test Case" be divided into specific pages (or views), allowing the user to define actions to be taken at each view. The user could then combine the various views and the entity-based scenarios into complete test cases. Shortly after, version 2.0 of the tool was released, consisting mainly of modifications implementing those two suggestions.
