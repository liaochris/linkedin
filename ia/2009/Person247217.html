  • PhD Student

    EDF R&D

    (Public Company; EDF; Oil & Energy industry)

    October 2008 — Present (9 months)

    Due to the increasing complexity of hardware architectures, making efficient scientific code is becoming harder. Nowadays, a developer in the scientific world needs many skills. For example, to make an efficient nuclear reactor core simulator, one need to know neutron physics, mathematics, computational science, computer science and computer architeture. In each of these fields, knowledge increases and choices become more complex.
    My research aims at finding a way for scientists to develop their code without any consideration to the computer architecture. This way should allow to preserve the performance of the codes accros the different target architecures (e.g. graphics card, multicore processors, PC clusters).
    To achieve this goal, I study the opportunites given by C++ templates to generate efficient code specialized for each targeted architecture.
