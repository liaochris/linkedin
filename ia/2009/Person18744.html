Avery Pennarun’s Summary

I didn't set out to be a technology entrepreneur, but after filling out the entries in my profile here, it appears that's what I am. My specialty is taking business ideas and turning them into working software products.

Avery Pennarun’s Specialties:

Programming (C, C++, C#, Python, Java, Perl, x86/68k/PPC assembly, SQL, Delphi, and others), Operating Systems (Linux, MacOS, Windows), low-level driver design, network protocol design, embedded systems, APIs, and user interfaces. For some reason, I really enjoy making automated software installers.

  • Co-founder and Chief Scientist

    Net Integration Technologies, Inc.

    (Computer Networking industry)

    1998 — 2006 (8 years)

    I co-founded Net Integration Technologies (Nitix) while I was still in university. Nitix is an all-in-one (hardware+software) network server appliance for small businesses. I was the lead developer for most of its evolution.

  • Developer

    Debian

    (Non-Profit; Computer Software industry)

    1996 — 2003 (7 years)

    My most interesting contribution to Debian was the Popularity Contest (popcon), which automatically collects information about software package popularity clusters. Debian (and now Ubuntu) can use these statistics to decide which packages are important enough to squeeze onto their install CDs. This takes a "typical" installation requirement from 3-4 CDs down to one, which is why you can always install Linux from a single CD image nowadays.

  • Software Contractor

    Bycast Inc.

    (Privately Held; Computer Software industry)

    2000 — 2000 (less than a year)

    Bycast was founded by one of my friends from high school, David Slik. I was hired to bring their first product, a highly reliable Internet audio/video streaming service, up to a "demoable" level.

    As part of the project, I specified, designed, and built an FPGA-based microsecond-precision time synchronization system using Global Positioning System technology, synchronizing incoming media samples to GPS time so that multiple network-based retransmitter devices could coordinate their signals without ever talking to each other.

    The result was Internet-based, real-time streaming media that was essentially "unkillable" - even if multiple network nodes (transmitters and routers) failed, your sound and video wouldn't suffer the slightest glitch.

    Bycast has gone on to apply the same technology to medical imaging systems.

  • Software Developer (Student)

    Mitel Semiconductor

    (Privately Held; Telecommunications industry)

    1999 — 1999 (less than a year)

    At Mitel Semiconductor, my job was to improve the device drivers for high-speed semiconductor test equipment costing upward of $1mil. Operators on the night shift would often become so confused by the software that an entire night's worth of testing would be aborted. I identified the main software problem areas and streamlined the workflow, resulting in fewer lost testing hours.

  • Software Developer (Student)

    General Datacomm

    (Public Company; GDC; Telecommunications industry)

    1997 — 1998 (1 year)

    General Datacomm's Multimedia Research Centre (MRC) was developing an enterprise-scale network videoconferencing appliance. My job was to work with a hardware engineer to bring up their embedded operating system working with his experimental new PowerPC-based motherboard design.

    I often tell people that the most fun part of this job was writing the very first few instructions that the PowerPC would execute on startup. They were all NOP. Anything else didn't work because our prototype power supply was so unstable.
