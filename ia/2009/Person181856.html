Patrick Corcoran’s Summary

Expert client-side developer for web-based software.

I have been making "Ajax" applications since 1999, long before the industry had a name for such things. The first Ajax-like project I worked on didn't use XMLHTTP, because that didn't exist yet, so it implemented its own transport vehicle using zero-pixel frames. It also did not use XML, since the specification for XML wasn't even at a final 1.0 version yet, so instead it just sent raw script and html back to the client. But the overall behaviorial patterns were undisputably Ajaxian.

Then in 2000, Microsoft's MSXML ActiveX plug-in to their 5.0 browser opened the door for true Ajax. In that year, I implemented a true Ajax interface for a financial services client in Manhattan who wanted to deliver a richer UI to their high net worth customers.

In 2001, I wanted to bring Ajax-like behavior to the Netscape 6.0 family of browsers. But since the Mozilla project had yet to implement XMLHTTP, I had to write an XML parser for the browser using JavaScript. The parser was written in about 500 lines of code and implemented most of the XML 1.0 specification. But more importantly it was performant enough to get the job done, allowing non-Microsoft browsers the ability to parse XML coming back from the client years before they were able to do so natively.
