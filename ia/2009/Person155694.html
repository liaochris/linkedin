  • Engineer

    The Athena Group

    (Computer Hardware industry)

    October 2006 — December 2008 (2 years 3 months)

    The Athena Group primarily develops IP for hardware accelerated security (encryption) and Digital Signal Processing such as the ubiquitous Fast Fourier Transform. These technologies come together in the form of a SoC (System on a Chip) design, utilizing microprocessors, periphery, and communication paths (buses).
    Following are highlights of my work at Athena:

    1. Property Specification Language. I was charged with the initial rollout of PSL (http://www.eda.org/vfv/docs/PSL-v1.1.pdf). When properly asserted, the language is used in the verification of Verilog and VHDL designs. This task was an expedient way for me to get my head back in the digital hardware design game.

    2. Bus Protocol Implementation. Over the course of about six months, I developed VHDL code to implement the AXI (AMBA 3, http://www.arm.com/products/solutions/AMBAHomePage.html)
    protocol. My design read/wrote on every clock cycle on all five channels making it an extremely efficient design and demonstrating that I could shorten typically long, arduous learning curves.

    3. Various VHDL design work. I cannot be specific about the designs, of course, but this task included both original architectures and debugging/extending existing designs. Domains included fully synchronous to latch-based and ranged in complexity from single files to upwards of a hundred. Technologies used : Modelsim, MatLab, Xilinx ISE 9+, VCS, ARM926EJ-S baseboard and corresponding tiles, ARM RealView, some synthesis tools, Agilent
    16800 LA/PG and InfiniiVision Digital Oscilliscope, and LabView to name some.

    4. Various software design work. Most software at Athena was C based and run on Unix-based machines. Some, however, was firmware targeted to the ARM 9 processor and run on the ARM926EJ-S baseboard.

    5. DSP related projects. I had the opportunity to help verify/design Athena's software controlled radio for the last eight months of my employ.

  • Software Engineer

    Morris Consulting Services

    (Computer Software industry)

    September 2004 — May 2006 (1 year 9 months)

    I worked for MCS twice for a combined total of just over one and a quarter years. During that time I experienced :

    1. The 1+ and 2+ .NET frameworks using VB and C# as primary languages.

    2. .NET CE for mobile devices. This was mostly maintenance and extension of existing code. As it was before the time of a readily available compact data store, a major component of the mobile system was the development of an XML database. Otherwise, the project entailed development on the VS emulators.

    3. Web services. When the mobile devices we docked, they uploaded their XML data via SOAP Web Services written in VB. This made our architecture four tier in nature : UI -> Business Logic -> Web Services -> Data store (MS SQL Server).

    4. Javascript. I wrote a website which heavily relied on Javascript for it's 'app-like' look and feel. Overcoming many browser dependencies and state issues was key. Today, however, Flash or at minimum AJAX would be more appropriate. Regardless, it was a wonderful learning experience.

    5. ASP .NET. In my second iteration with MCS, we relied heavily on the 2.0 version of ASP .NET's functionality such as provider models, validation objects, login tool etc. Concerning C#, generics and other 2+ improvements were also used.

    6. Legacy work. Technologies such as J++ and maintenace of older versions of SQL Server were required.
