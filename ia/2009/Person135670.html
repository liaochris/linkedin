Kurt Newman’s Summary

"The first 95% of software development is easy, it's the last 95% that's hard."

My goal is to create environments that make the last 95% easier.

Thus, one of my main goals related to software and security is to find efficient mechanisms for accomplishing a task that reduces future workload in the form of support and maintenance.

I'm currently working on CISSP certification.

Kurt Newman’s Experience

  • Software Engineer

    Global DataGuard

    (Privately Held; Computer & Network Security industry)

    January 2003 — Present (6 years 8 months)

That said, I've been responsible for architecture, design, implementation, and unit-testing while supporting an entire life-cycle of existing work. It's expected that I move projects to completion by working as a team with little to no supervision, and to ensure specifications are being met for the customer.

    Some of my past projects:

  • Security Analyst

    Global DataGuard

    (Privately Held; Computer & Network Security industry)

    December 2001 — January 2003 (1 year 2 months)

    The focus of this position was to ensure network integrity of our customer networks by using Global DataGuard's proprietary IDS systems and tools. It required technical proficiency, co-worker and group discussions about remediation, as well as the ability to interface with customers if/when events occurred.

    The technical skills needed were as follows:
    - Perl and shell scripting
    - Analyze and look for erroneous/iniquitous behavior by using a deep understanding of Internet protocols
    - Ability to function in a Linux environment by using shell scripts and ad-hoc strings of commands
    - Proficient use of tcpdump for raw packet analysis
    - Configuration of Dragon IDS, a signature intrusion detection system (NIDS)
    - Understanding and interpreting the results from Global DataGuard's proprietary behavioral network intrusion detection system (NIDS).
