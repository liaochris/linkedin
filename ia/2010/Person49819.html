  • Architect, designer, and implementer

    Independent Project

    (Computer Software industry)

    2005 — Present (5 years )

    Architected and implemented a stand-alone digital audio appliance that can be connected directly to a home audio system and controlled by network device that can speak XML/HTTP.

    This project has included:
    • Writing an internal C-language API to control the audio player and manage the music database and playlist.
    • Writing an XML/HTTP interface on top of the C API.
    • Writing an AJAX style web interface that runs on top of the XML interface.
    • Writing a component to manage audio files from different sources so that the appliance can provide a cohesive view of the entire music library, enable shuffle-play, searching, etc.
    • Integrating a number of open-source components, while keeping the footprint small (all of the software, including operating system and applications fits into a runtime image that is about 5MB). This project has evolved since its inception, and at one time it used a small Linux distribution that I built from the ground up, but I’ve since decided to leverage uClinux-dist along with a number of other 3rd party packages: Busybox, mpg123, uClibc, Apple’s “Bonjour” (multicast DNS), Mini-XML, and the Boa web-server.

    Since I architected all layers of this system, it has provided me with an opportunity to gain experience with a wide variety of technologies and open-source software packages. I hope to soon publish this as an open source project, which will hopefully entice others to write new user-interfaces and add new features.
