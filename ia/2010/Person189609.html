Smart server procurement arch RISC/CISC 1) Environmental impact
Asking questions - Does the vendor have a WEEE (Product Take Back) program? Are the electronics RoHS compliant? How can we reduce server sprawl? What is the data/application life cycle?
The "old" web app env 1 server = 1 application model - database / application / web front = underutilized resources / isolated environments, no framework in place to even gauge hardware/software bottlenecks = throw more hardware at ANY issue..
Consolidate on 4/8 socket quad core / multi-thread systems, technology like SUN's UltraSPARC T2 "system on a chip". Use virtualization SUN's zones / LDoms (free) / Linux xen (free) / Swsoft's virtuozzo / Vmware's ESX..
Cost & support - Be smart.. purchase used Cisco gear not under a smartnet contract... Get a chassis 6500 chassis under smartnet - no need to put module cards under smartnet.. Look for "Factory Refurbished". Cisco TAC superb reputation engineering support. Use a mix for multiples, hot spares onsite plus smartnet. Stackable solution trunking ports instead of the chassis. Look for direct purchase programs like SUN's partner advantage programs. Why not signup as a reseller? And purchase NFR products...
More and more manufactures are selling direct. Why pay double the cost for a Chatsworth cabinet - purchase direct from Rack Solutions.
Storage - Do you homework.. No reason to go clariion.. HBA to the SAN active/active) SUN's Multiplexed I/O (MPxIO) and Linux (DM Multipath) multipath IO
