Lale Divringi’s Summary

My passion is building cutting edge technology and software systems, specializing in mobile programming, web services, Relational Database / Data Access solutions.

A senior technologist, I have worked on building a number of successful products for Microsoft. I have been a member of the SQL Server Dev Team since the very early days, working on and amassing in-depth knowledge of MS SQL Server and data access components such as ADO.Net systems.

I also have 3+ years of experience building business solutions on the mobile platforms.

Lale Divringi’s Specialties:

I am an experienced, hands-on technical professional. I have led teams building solutions based on ADO.Net, .Net Frameworks, XML Access, SQL Server, IIS ISAPI, SOAP, HTTP-DAV and similar technologies using C# and C++.

Lale Divringi’s Experience

  • Director of Software Development

    Formotus

    (Privately Held; Computer Software industry)

    March 2007 — Present (3 years 5 months)

    Developing software plus services for deploying electronic forms to mobile devices. Formotus back end server is an asp.net application using SQL Server 2005 as storage to manage organizations, users and forms. Formotus client application runs on Windows Mobile and Google's Android devices, allowing users to fill in and submit the forms out in the field. Data submitted through Formotus mobile client is available for view either through desktop or through the mobile device.

  • Software Development Lead

    Microsoft - SQL Server-.Net Frameworks ADO.Net

    (Public Company; 10,001 or more employees; MSFT; Computer Software industry)

    March 2001 — January 2007 (5 years 11 months)

    Led a team of up to 8 Software Development Engineers to develop the ADO.Net component for .Net Frameworks and SQL Server. ADO.Net is the data access platform for .Net applications, consisting of two major components: DataSet (a disconnected in memory database cache) and Managed Providers (SqlClient, OracleClient, OleDbClient and OdbcClient). My team worked on shaping the strategy and vision of Microsoft’s data access platform. In addition to providing technical leadership to my team, I managed cross group collaborations in order to meet the data access needs of other divisions within Microsoft (like Office, MSN, WinFS), ensuring and meeting the quality gates for development and shipping of the product.

Also implemented XML based system stored procedures for SQL Server 2000. Personally designed and implemented updategrams, a means to modify SQL Server data based on XML format and annotated schema.
