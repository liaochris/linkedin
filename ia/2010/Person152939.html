  • Senior Developer / Technical Architect

    Enviros

    (Privately Held; 201-500 employees; Environmental Services industry)

    January 2004 — November 2007 (3 years 11 months)

    I was the technical architect and lead developer for a product called Montage. Developed using n-tier design being able to scale up and down appropriately. At our largest client we were processing around 3 million complex energy management formulas every hour.
    The data extraction part of the system runs under a Windows Service making full advantage of multi-threading capabilities to increase performance. Using a web-based front engine management tool the users can flexibly obtain data held in almost any format from multiple sources such as SQL server, Oracle, SAP, PI, WebServer, Web page or any ODBC compliant datasource.
    The processing engine was also a multi-threaded Windows Service that used to process millions of transactions every day.
    The web-based reporting and management portion of the system was fully AJAX enabled with extensive use of CSS.
    The product was utilised Visual Studio 2005 Team Suite integrated with Team Foundation Server using TeamBuild and Installshield.

  • Senior Developer / Technical Architect

    VirtualBackup Incorpated

    (Privately Held; 11-50 employees; Information Technology and Services industry)

    April 2002 — March 2004 (2 years )

    I developed a global off-site backup solution which is live on 4 continents. The system currently securely maintains multiple load-balanced servers around the world that holds data for tens of thousands of users.
    The customer administration system was developed using ASP.NET and C# components talking to a SQL Server 2000 backend database. The financial administration system was also created using the same technologies. The two systems interacted via webservices developed using Visual Studio.net.
    The client software was a self-maintaining .net based windows form application. It interacts with regional and central servers via webservices and winsock communications, all custom developed in c#.
    All data and communications are heavily encrypted using the .net framework based encryption technologies.

  • Lead Developer / Technical Architect

    First Class Group Limited

    (Privately Held; 11-50 employees; Mechanical or Industrial Engineering industry)

    January 2002 — April 2002 (4 months)

    This company installs and maintains the shop fronts and shutter doors for companies such as Boots, J.J.B.’s and Woolworth’s. I was asked to develop a new browser-based system to manage their service and callout requirements.
    The system runs under IIS on Windows 2000 utilities the Microsoft .net framework with a SQL Server 2000 data store. The presentation components were written in C# and ADO.NET with the presentation code provided by ASP.NET and C#. Third party companies such as Boots have capability to access XML Web Services on the site as well as remotely logging new calls into the system.
    Developed under a DSDM methodology, the development went very well with future enhancements of the project looking at integrating the system with Microsoft PocketPC based systems in the engineers vans and cars.

  • Senior Developer

    HM Customs & Excise

    (Government Agency; 10,001+ employees; Government Administration industry)

    June 2001 — January 2002 (8 months)

    I developed the Debt Management systems for the HMCE. This involved web based systems which communicated with Traders, Solicitors, Bailiffs and Insolvency Practitioners. This could with be via a Web Interface or a B2B style XML Posting system using the Government Gateway. The system was based on Microsoft Technologies but with a need to communicate to the main VAT database. It was a distributed platform with each layer physically separated on different clustered environments separated via firewalls. Communication between the layers was using XML via SOAP and/or DCOM.
    The User Services run under IIS on Windows 2000. ASP pages execute methods on VB COM+ components, these methods return XML which is then transformed into XHTML via XSL and the Microsoft XML Parser. The Business Services Layer is comprised entirely of VB COM+ components running in a Windows 2000 environment. The Data Services Layer consists of Visual Basic COM+ components that talk to SQL Server 2000 databases via ADO.

  • Lead Developer (Contract)

    Comax Secure Business Systems

    (Privately Held; 201-500 employees; Defense & Space industry)

    October 1999 — December 1999 (3 months)

    Recruited to solely design and develop secure on-line sales and purchase ordering system for the IT Procurement for the Ministry of Defence (DERA).
    The User Services Layer utilises a Visual Basic front end and an Internet Explorer front-end, designed for use in E-Commerce applications. The browser portion allowed for shopping trolley-style applications for ordering over a secure Intranet. This communicated via ASP to a central component within the User Services architecture. The VB front-end used ActiveX controls, dynamically added, to allow for ease of maintenance.
    The Business Services Layer ran a series of state-less Server-side objects under the control of Microsoft Transaction Server. It held all of the business rules and provided transactional processing for all of the business requests.
    The Data Services Layer connected via ADO to SQL Server 6.5 and 7.0 databases. The database comprised of about 30 tables and has heavy usage of stored procedures and custom views.
