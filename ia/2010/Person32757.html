  • Principal Diagnostic Engineer

    Airvana

    (Public Company; AIRV; Telecommunications industry)

    April 2001 — March 2010 (9 years )

    Wrote diagnostics for a CDMA Femtocell. This Femtocell contained both 1xRTT and 1xEV-DO Rev-A radios. Also wrote the Hardware Abstraction layer for the RF engineers which allowed them to program and calibrate all radio devices and perform rapid qualification testing using the diags. Including utilities for all high level radio functions such as turning on/off each of the radios, set the channel/frequency, set output power, turn AGC on/off, etc.

    Worked with the Platform group to develop drivers for the 1xEV-DO radio. The radio utilities I developed were ported to the application code to operate the radios in the CDMA Femtocell.

    Designed and developed diagnostics for a UMTS based Femtocell. Also wrote command line utilities for the RF engineers to allow them to program and calibrate all radio devices. With these utilities, the engineer could turn on/off each of the three radios, set the channel/frequency, set output power, etc.

    Led a team of three diagnostic engineers on the SkyBTS Base Station, which is a 1xEV-DO Rev-A based Air-to-Ground communication system. Architected, designed and co-developed FLASH & RAM based Power-up and Offline Diagnostics. Tests were developed for: DDR, I2C, SPI, EEPROM, T1/E1, RTC, FPGAs, etc. Project used the Qualcomm CSM6800 and had three Radios. The radios where over 50% analog and required Initialization and setup utilities to allow the engineers to test the units before the software was ready. Also ported DSP tests from Qualcomm for use in our diagnostics.

  • Principal Diagnostic Engineer

    June 1999 — April 2001 (1 year 11 months)

    Designed and developed FLASH based Power-up and FLASH based Offline Diagnostics in C & C++ for the PMC-MACH48 on the OZ project, which was an OC48 multi-service access device on a channelized interface.

  • Principal Diagnostic Engineer

    BayNetworks / Nortel Networks

    (Public Company; NRTLQ; Telecommunications industry)

    September 1997 — June 1999 (1 year 10 months)

    Designed and developed EPROM based Power-up and FLASH based Offline Diagnostics for the ATM Trunk card on the Versalar 15000. Diagnostic tests were developed for the Maker MXT3010 SAR controller chip. MPC801 based tests were also developed for memory, interrupt generation, registers, communicating with the SARs and messaging across the I2C bus to Serial E2PROM devices. Designed and developed tests for the Utopia II interface to several PHY devices, i.e., OC48. Designed and developed FLASH based Offline Diagnostics for various PHY cards, such as: OC12/48, DS3, OC192, etc.

  • Principal Diagnostic Engineer

    Cascade Communications

    (Public Company; CAE; Machinery industry)

    April 1992 — June 1997 (5 years 3 months)

    Only Diag Engineer for first 2 years at Cascade.

    Architected, designed and developed EEPROM based Power-up and FLASH based Menu driven Offline Diagnostics for the B-STDX 9000 Frame Relay switch. Diagnostic tests were developed for Low & High Speed Cores, Control Processor, T1/E1, HSSI and V.35 I/O Controller cards. Worked with driver developers to develop drivers for various cards.

  • Principal Diagnostic Software Engineer

    Wang Laboratories

    (Public Company; WANG; Computer Software industry)

    August 1983 — August 1991 (8 years 1 month)

    Project leader for diagnostics software and prototype test software for all 802.3, 802.5 and FDDI products. Designed and implemented diagnostics for intelligent IBM PC network cards, Motorola 68020 based 802.3 bridge and Intel 80186/80386 based 802.3 and 802.5 and FDDI I/O controllers. Member of FDDI controller design team.

    Designed and developed a PC based stand-alone 802.3 LAN Exerciser/Diagnostic tool. This application program has been used by R&D to stress and diagnose faults in all 802.3 hardware and communications software developed at Wang (PC network cards, minicomputer I/O controllers, bridges, repeaters and encryption units). It is used by field support personnel and customers for network analysis, fault detection and post repair testing.
