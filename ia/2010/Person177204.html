Ron Gauthier’s Summary

Founded two successful companies in the engineering services industry: Sysdyne Corporation and EXB Solutions.

Over 30 years of experience as a systems/software design and test engineer working as a team leader or member. Comprehensive experience with full life cycle software development including requirements specification, design, implementation, debug, test, and documentation. Experienced in formal software development using standards such as FDA (medical), DO-178A (commercial aviation), and DOD-STD-2167A & MIL-STD-498 (military). Also comfortable with project planning, budgeting, scheduling, customer interfacing, and training. Knowledgeable in using a variety of hardware and software development tools including IDEs, debuggers, in-circuit emulators, logic analyzers, profilers, configuration management tools, issue tracking tools, and scripts.

Industries served include medical, military, commercial, avionics, computer security, and others. Applications involve real-time embedded software, hardware/software integration, end-user applications, dynamic simulation and control, state machines, mathematical modeling, data acquisition, numerical analysis, test procedures, automated test tools, and software tools.

Ron Gauthier’s Experience

  • Founder / Owner / Director

    EXB Solutions

    (Aviation & Aerospace industry)

    2000 — Present (10 years )

    EXB Solutions, Inc. is an engineering services business focused on systems and software test for high-reliability and safety-critical systems in the aerospace and defense industries. EXB’s systems and software test services are performed as part of development or as an independent verification and validation effort. EXB further focuses its test services on guidance and navigation, flight controls, flight management systems, and sensing technologies, and commonly works in real-time embedded engineering environments.

    Boston Scientific / Guidant Corporation

  • Software Engineer

    Eastman Kodak / Imation

    (Medical Devices industry)

    1997 — 2005 (8 years )

    Health Imaging Division. Kodak acquired Imation's Health Imaging Division in late 1998.

    Software design engineer on a team that developed three different laser imagers (FDA regulated imaging devices for printing diagnostic quality medical films). Developed in compliance with FDA software standards.

    Responsible for the full life cycle development of the software managing the cartridge controller subsystem, a custom controller running an 8051 compatible microprocessor. This subsystem is responsible for opening/closing film cartridges, picking up film, and initiating film transport for three film drawers concurrently. The real-time embedded software manages a total of 18 motors (DC and stepper) and 48 sensors, and supports encrypted in-application flash reprogramming. The multitasking software is built on a commercial RTOS (RTX51) and uses an I2C serial bus as the internal communication link with the rest of the system. Primary development tools were the Keil 8051 toolset and µVision IDE, the Philips PDS51 in-circuit emulator, and standard test equipment such as oscilloscopes. Responsible for hardware/software integration and technical interface to both the electronics and mechanical teams. Assisted the electronics engineers on board design improvements. Helped design a test device that simulates all motors and sensors, including all failure modes, for subsystem testing.

    Designed and implemented several components of the Image Management System real-time embedded software, including the controllers for two high speed PCI video capture boards. The target platform was an embedded Motorola MTX computer system running pSOS+, a real-time, multitasking, multithreaded, operating system kernel. The object-oriented software design was implemented in C++ using Diab Data and GNU tools on Unix workstations. Primary technical interface with the video board vendor. Developed and conducted training sessions on the video subsystem.

  • Software Engineer

    Secure Computing Corporation

    (Public Company; SCUR; Computer & Network Security industry)

    1993 — 1996 (3 years )

    Software design engineer for a Secure Network Server developed to provide policy controlled information exchange between multiple classified nodes (secret, top secret, etc.) and insecure networks. Member of a small team that developed a controller for encrypting all data residing on SCSI storage devices as well as providing coprocessor cryptographic services to the host system. The controller was implemented using a multiprocessor Motorola MVME162 board running custom OS kernel embedded firmware. Responsible for the full life cycle development of 3 major components of the OS firmware, which was implemented using C on Sun workstations running Unix. All design and documentation was done using the DOD-STD-2167A standard.

    Developed a set of CryptoCard API routines for use by Sidewinder, a technically sophisticated network firewall. The CryptoCard is a PC (PCMCIA) card used for the identification and authentication of network users.

  • Software Engineer

    MTS Systems

    (Mechanical or Industrial Engineering industry)

    1992 — 1993 (1 year )

    Software design engineer responsible for full life cycle development of several major components of the Automated Site Controller (ASC) portion of an advanced road simulator test system. The system provides controlled road simulation in a lab for analyzing full size automobile road test vehicle dynamics. The ASC is a configurable, high performance, real-time controller capable of high speed data acquisition, drive (playback), and analysis for large channel counts. The ASC contains multiple embedded processors including a Sparc board running Unix, Motorola 147 or 167 processor boards running pSOS+ (multiprocessor version), and a proprietary board running digital signal processors. Responsibilities included the inter-processor message dispatch system, buffered file and data servers, device drivers, memory management, error handling using exceptions, and some tool development supporting multiple targets. All software was developed on Sun workstations using C, object-oriented techniques, and a proprietary object-oriented programming system.

Responsible for design specifications, cost quotes, schedules, system design, software design and implementation, directing tester build, and interfacing with customers and support groups. Software developed using DO-187A commercial aviation standards. Installed test stations at the MacDonnell Douglas Long Beach facility and trained technicians to use the stations.

    Designed and implemented a Global Positioning System Sensor Unit simulator. This is a PC based, real-time simulation written in Pascal that makes it possible to test devices which incorporate GPS functions without relying on the actual GPS satellites. Responsible for researching vendors and selecting the hardware used in the simulator.
