  • Development Manager/Sr Software Engineer

    Witness Systems, Inc

    (Computer Software industry)

    September 1998 — December 2004 (6 years 4 months)

    * Managed team of three engineers
    * Designed and developed a common presentation framework for use by other development teams within the company. It was designed and developed using the MVC design pattern. The framework allowed developers to quickly build a user interface with a common look and feel and a common interface to business services, cutting development time. The user interface was created using XForms and cascading style sheets. A state diagram representing the flow of the application was then developed in Control Center. After the business services are created, the appropriate method calls were entered into the state diagram as transition actions. The state diagram was then exported as an XML file via a Control Center plug-in. The framework used cocoon pipelines to transform the XML to and from the user. Those transformations were in the form of XSL style sheets and custom Java transformations. The last transformation was an internationalization transformation that allowed us to support multiple languages. The framework interfaced with business services on the same machine as the framework. This was accomplished via native Java calls. The services could also be on a remote machine and link to the framework via SOAP calls. The initial design of the application used use cases as well as class, state, sequence and deployment diagrams.
    * Designed and developed a web-server monitoring product that allowed our customers to capture and replay web-site interactions. The customers could see what pages and flows gave the users trouble. The product linked into both IIS on a Windows platform and Apache on a Sun UNIX platform.
