Burkhart Wolff’s Summary

My research interests are focussed on Software Verification Environments. This covers top-down refinement methods (like HOL-Z or HOL-OCL), bottum-up code verification methods (like IMP++ based on HOL-OCL or HOL-Boogie) or specification-based testing methods (like HOL-TestGen). I consider myself as a "Formal Methodist" in Software Engineering.

Application areas of these techniques have been:
- specification languages (CSP,Z,UML/OCL, IMP++)
- computer security (CVS-Server, Hitachi's DARMA)
- computer architecture, operating systems (Microsoft's Hypervisor, recently)

I work with interactive as well as automated theorem provers, I work as a tool developer, and yes, I'm still programming myself.

In the late nineties, I developed a proof-environment for infinite CSP (HOL-CSP), later the HOL-Z environment (used for substantial case studies in computer security) and most recently HOL-OCL.

At the ETH, I co-authored the HOL-TestGen system that converts a test theory in HOL together with a test-specification semi-automatically into test-driver for the system - under - test running generated test-data. HOL-TestGen has been applied for very substantial tests of network filters (firewalls).

Within the VerisoftXT project, I developped also a proof environment for "real" C, i.e. the C used in the Microsoft Hypervisor, called HOL-Boogie. Technically, it is an interactive theorem proving backend for VCC-I (and II).

At the University of Paris-Sud, I will work on formal
testing techniques along HOL-TestGen, aiming at higher
performance as well as support for a wider scope of testing techniques and their combination with theorem proving techniques.
