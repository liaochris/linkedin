Rick Mouritzen’s Summary

“Simplicity is the ultimate sophistication.”
-Leonardo da Vinci

I'm an independent software consultant specializing in complex data backed web applications. My current favorite language for this task is Clojure, though I have used various Java technologies since 2000.

I've been working in software development professionally since 1991. Most of this time has been hands-on, though I managed for two years while at Symantec.

I've spent considerable time dealing with performance, scaling and concurrency issues. My emphasis is creating simple solutions to complex problems. (It's also one of my frustrations working directly for large corporations. Complex solutions require Herculean effort are richly rewarded. Simple solutions are trivialized.)

My goals of simplicity, elegance and sophistication are why today I am using a dialect of Lisp in my own projects.

I've spent considerable time with relational databases. I've also been part of the NoSQL movement, having used non-SQL solutions in most of my recent work. Unlike with the EJBs (there's a good use of EJB, but your use-case is almost certainly not one of them) there are times when an SQL database is the right solution. Further when doing NoSQL there are times when you want ACID and times when you do not (or cannot).

I've spent quite a bit of time dealing with and thinking about concurrency and threading issues. It is possible to build to build simple concurrent systems using Java and the Windows API (two technologies I am familiar with), though in my experience it is error prone and requires too much convention (contracts not explicitly stated or enforced in code). Erlang style actors and Clojore's immutability along with software transaction memory (STM) are the path forward. Simplicity is the theme.

Rick Mouritzen’s Specialties:

Web application development of data backed applications.

Most of this work has used Java. In the last year I've done a deep dive on Scala, Objective-C, Common Lisp and Clojure. I've spent enough time with Python (with Django) and Ruby (with Ruby on Rails) to know I prefer them over Java. I use PHP just enough to tweak my blog.

I'm also quite familiar with C and C++, having spent the first 8 years of my career using those languages. Most of this work targeted Windows.
