Benjamin Habegger’s Summary

I obtained a Phd in Computer Science from the University of Nantes on the topic of Information Extraction from the Web. During this time I developed (particularly in perl) two extraction tools. First, IERel allows to learn string-based extraction patterns from a given set of example labeled web pages. Second, Websource automates web navigation by "executing" a web navigation workflow described in and XML format.

After my thesis, I worked at the University of Lille on machine-learning XPath-like queries using dynamic programming techniques. A prototype was developed in Ocaml.

In parallel to my research I have given courses on many different subjects in computer science among which database technologies, functional programming, logics, formal languages, XML technologies, and others.

After Lille I worked for 8 months at the University of Rome on the topic of relational machine learning for data integration. There, I developed tool in Java allowing to machine-learn mappings (essentially queries) between a database and an ontology. This tool has a user-interface using the Mozilla/Gecko XUL technologies. During this time I also did some work on defining a task-oriented user desktop.

My research years have lead to a number publications on learning extraction and integration patterns in different data and query representations : strings, trees and relational. I also regularly participate doing reviews for conferences or journals.

I am currently working at Nirva Systems Ltd., a software company offering and application integration platform (Nirva) specialized in document processes and mass printing industry. Nirva is a service oriented and command based platform. It rapidly allows to build specialized applications using light weight interfaces accessible via its embedded HTTP server. Applications in Nirva widely use XML and XSLT. Building application in Nirva can make use of many languages among which Java and Perl.
