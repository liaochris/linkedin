  • Senior Programmer

    Narus

    (Privately Held; Computer Software industry)

    December 2006 — March 2009 (2 years 4 months)

    Fixed bugs in the first-generation programmable deep packet inspection parser of the Narus Semantic Traffic Analyzer. Designed the second-generation programmable DPI parser and its programming language, and implemented the system with a local coworker. Hand-coded the core regex loop in assembly and optimized it to 8 instructions. Second-generation DPI parser is about 10x faster than the first-generation, and uses less than half the RAM.

    Designed and implemented a second-generation generation back-end processing layer for the aforementioned DPI system. Processing layer was implemented as a custom concurrent message-passing (actor model) programming language which automatically executed the user program across multiple cores. Co-wrote the compiler (source-to-C translator), runtime library, and message-passing kernel for the language implementation with three teammates at Indian office. This second-generation back-end processing layer was 5.7x faster than the previous version (3.8x faster per CPU at 150% CPU usage) for the first ported application.

  • Consultant

    Sony Computer Entertainment

    (Public Company; SNE; Consumer Electronics industry)

    October 2005 — February 2006 (5 months)

    Analyzed a graphics interpreter for optimization opportunities. Designed and coded a just-in-time compiler (JIT) for debugging purposes (similar to valgrind). This JIT converted MIPS code into a virtual code, inserted instrumentation code for detecting out-of-bounds memory accesses, then converted the virtual code back to native MIPS code on either a basic block or per-function basis. It also maintained a translated code cache to avoid retranslation. The dynamically translated code ran at about 25% of original code speed while detecting out-of-bounds memory accesses.
