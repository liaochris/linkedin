  • Software engineer

    (Government Agency; 5001-10,000 employees; Computer Software industry)

    August 2005 — September 2006 (1 year 2 months)

    Java programmer in FESA project (Accelerators and Beams department).
    FESA is a framework to design, create, deploy and monitor real-time software controlling beam accelerators' equipment.

    I was responsible for maintaining and extending design tools. One of my most interesting long-term tasks was to find a flexible way of defining complex XML constraints (XML-schema appeared to be not sufficient).
    After analyzing possible and available solutions, I created a Java native implementation of CLiXML language which is a mixture of XPath and first order logic. Thanks to that, XML constraints could have been relatively easily implemented without expertise knowledge - basic understanding of XPath, XML and first order logic was enough in most cases.
