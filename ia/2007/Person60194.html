Miska Hiltunen’s Summary

Embedded software experience since 1993.
Software quality as special interest area. Pragmatic approach focusing on error prevention instead of correction, favoring more effort on code inspection instead of testing, preferring regular and routine checks to big and rare inspection rituals.
Developed a pragmatic, back-to-the-basics code inspection method that helps companies utilize the preventive edge in their software development.

The method is called Tick-the-Code Inspection.

I find the writings of Stephen R. Covey, Gerald M. Weinberg and Tom Gilb especially inspiring.

Miska Hiltunen’s Specialties:

Qualiteers offers half-a-day in-house Tick-the-Code training to programmers around the world. See http://www.qualiteers.com/.
Error prevention and code inspection have never been combined quite like this.
