Ronald Mintz’s Experience

  • Sr. Programmer Analyst

    First Consulting Group at New York Presbyterian Hospital

    (Public Company; 1001-5000 employees; Hospital & Health Care industry)

    June 1998 — November 2005 (7 years 6 months)

    • Designed and developed clinical alert programs to support and alert physicians and other users of electronic patient chart provided by Eclipsys Sunrise Critical Care. Wrote programs in C with RDB (relational database) in UNIX environment.
    • Wrote UNIX RDB scripts to transform source data tables of clinical alert parameters into the runtime database.
    • Together with my supervisor Dr. Michael Oppenheim, published description of techniques used by clinical alerts in proceedings of American Medical Informatics Association Symposium 2000 entitled "Design of a Clinical Alert System to Facilitate Development, Testing, Maintenance, and User-Specific Notification."

  • Sr. Programmer Analyst

    Popkin Software and Systems (now part of Telelogic)

    (Privately Held; 51-200 employees; Computer Software industry)

    1994 — 1998 (4 years)

    • Developed and enhanced portions of System Architect CASE tool using C, Visual C++, and Windows SDK. Solved software problems considered to be extremely difficult by technical managers and senior coworkers.
    • Enhanced and maintained Reverse Data Engineer to parse SQL files for Sybase, Oracle, Microsoft SQL Server, Informix, and DB2. Created data modeling diagrams and definitions in System Architect from these SQL files. Fixed serious bugs in the LALR parser generator and its error recovery routines. Significantly increased the parser’s tolerance of syntax variation.
    • Updated Schema Generator to produce DB2 SQL files from System Architect diagrams and definitions. Enhanced Schema Generator to accommodate Double-Byte Character Set (DBCS) to enable it to process Oriental languages.
    • Stored graphical diagrams in form of Windows metafiles. Implemented algebraic transformations on parameters of commands in these metafiles to position and scale diagrams in formatted reports.

    1993 — 1993 (less than a year)

    • Designed and developed real-time market data system in C to receive broadcasts from the Knight-Ridder Digital Data Feed and to maintain a real-time display of selected market data.
    • Developed quantitative algorithms for commodity and foreign exchange trading and implemented them in C programs. Developed C routines for graphical display of trading decisions and their results.

  • Sr. Programmer

    Bayer, Inc. (Diagnostics Division - formerly Technicon Instruments Corp.)

    (Public Company; Medical Devices industry)

    1985 — 1992 (7 years)

    • Developed complex data analysis programs and implemented numerical algorithms for calculation of medical laboratory test results. Worked with clinical chemists and system engineers. These programs were written in a well-structured, easily readable style and proved to be highly error-free.
    • Wrote real-time programs to control sequencing of calculations of medical test results. Developed test cases to validate the real-time performance of the system under an extreme workload.

  • Sr. Software Engineer

    Honeywell Information Systems (formerly Incoterm Corp)

    (Public Company; Computer Hardware industry)

    1977 — 1985 (8 years)

    • Designed and implemented data communication software to emulate cluster of VIP 7814 terminals. Created task structure to provide real-time emulation for multiple users. Trained other programmers in technical details of design.
    • Deduced undocumented principles of operation of Honeywell VIP 7700R and 7760 terminals. Described them for project team. Rewrote major portions of the real-time emulation software to more accurately emulate these terminals. Coding proved to be extremely error-free in field testing.
