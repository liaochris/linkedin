Each task is processed in a separate thread and all the tasks/threads are concurrently run using the Java multi-thread framework. Tasks include reading postings from the dmPOSTINGEVENT table, validating postings, re-mapping account numbers, netting balances, etc. Xml messages are generated and published to a proprietary dbus channel where the downstream apps subscribe to. The Java main program communicates with the C++ agent through TCP socket streaming using a proprietary protocol.
