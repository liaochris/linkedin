  • Freelance Analyst / Developer

    QXL

    (Public Company; 201-500 employees; Internet industry)

    July 1999 — December 1999 (6 months)

    Creation of a high-speed application server for QXL (QXL is an online auction company www.qxl.com).

    The main problem facing QXL was that it was unable to cope with the high Internet traffic it was receiving. At this time applications servers were still in their infancy and could not cope with loads of well over 20,000 concurrent users at peak times. I was required to design a system that could handle the large number of users the site was receiving and to make it flexible enough to allow the HTML developers the ability to quickly modify the website.

    The application server I designed and developed worked in a similar way to JSP except that the code embedded into the HTML was native C++ code. I developed a server that would translate the HTML/C++ files into C++ code and compile them “on the fly” into shared libraries. In addition I developed a multi-threaded server that could load these shared libraries and execute them to produce a HTML output.

  • Freelance Analyst / Designer / Developer

    Lloyds Bank Securities Services

    (Investment Banking industry)

    March 1998 — June 1999 (1 year 4 months)

    The system was an e-commerce project designed to allow secure trading and report delivery between Lloyds Bank Securities Services (LBSS) and their customers. My role was the initial design of the system and later to lead the implementation.

    The system was made up of three tiers. The first tier was a PC based application use for entering trades and receiving reports written in Visual C++ and Visual Basic. Network security was implemented with a mixture of secure sockets and smart cards used for authentication. The second tier was a middleware layer written in C++ on a HP-UX machine. The middleware was responsible for ensuring the validity and security of client connections. The third tier, also running HP-UX contained an Oracle database and ran the overnight batch processing modules.

  • Freelance Developer

    Lloyds Bank Securities Services

    (Investment Banking industry)

    April 1996 — October 1997 (1 year 7 months)

    I worked on the Lloyds LASER project. This is an in-house Global Custody's system written using Oracle 7, Pro*C and Designer/Developer 2000.

    My initial responsibilities within the project were to design and implement the accounting overnight batch processes. This consisted of batch jobs to calculate interest payments, standing orders, sweep accounts and jobs to handle backdated payments.

    Later I became responsible for the projects workflow system. The workflow system was made up of a number of UNIX daemon processes (written in Pro*C), which controlled the assignment of tasks to LASER users. My main responsibility within this area was to address Oracle performance issues.
