Satish Chandra Gupta’s Experience

  • Developer & Tech Lead, UML based Action Language in Rational modeling Tools

    IBM

    (Public Company; 10,001 or more employees; IBM; Information Technology and Services industry)

    March 2008 — Present (8 months)

    Rational has a family of UML modeling tools that includes Rational Software Architect (RSA http://www.ibm.com/developerworks/rational/products/rsa/) and Rational Systems Developer (RSD http://www.ibm.com/developerworks/rational/products/rsd/). These are Eclipse based tools used for Model Driven Development of C/C++ and Java applications. Currently most of the code is generated from UML model, but code for method bodies, and action code snippets in state machine must be written in the target language (C++, Java etc.). I am working on designing and implementing support for a UML based Action Language in these tools so that models are “purely” in UML, and can be simulated/debugged independently of the target language, especially in RSD-RT, which is used for designing and developing soft real-time concurrent reactive systems (such systems can be modeled using communicating state machines).

  • Developer & Tech Lead, Rational PurifyPlus for AIX

    IBM

    (Public Company; 10,001 or more employees; IBM; Computer Software industry)

    July 2005 — February 2008 (2 years 8 months)

    I worked as the Tech Lead for PurifyPlus/AIX. PurifyPlus consists of Purify (detects memory corruption errors), Quantify (finds performance bottlenecks) and PureCoverage (for code coverage analysis). These tools perform Object Code Insertion, and are tightly coupled with underlying compiler, OS and hardware.

    Roles: Tech lead for AIX/PowerPC, conceptualize new features & data/control flow analysis, design, development, testing, mentoring junior team members, evangelizing (inside and outside IBM) methodical use of PurifyPlus to improve software quality.

  • Developer & Tech Lead for Java Leak profiling in Rational Application Developer

    IBM

    (Public Company; 10,001 or more employees; IBM; Computer Software industry)

    December 2003 — June 2005 (1 year 7 months)

    I was the Tech Lead of the team in India responsible for productizing Java leak detection technology (Leakbot) developed at IBM research as a profiling capability in Rational Application Developer (RAD). The main components were: JVMPI data collection agent (C, Eclipse/TPTP), leak analyzer (Java, Eclipse plug-in based on Leakbot), GUI (Java, Eclipse plug-ins).

    Roles: Tech lead, design, development, testing

  • Research Assistant

    University of Wisconsin - Milwaukee

    (Educational Institution; 10,001 or more employees; Research industry)

    August 1999 — January 2002 (2 years 6 months)

    The Software Concordance is a prototype programming environment for experimenting with the benefits of enhanced hypermedia services for managing software documents, especially program source code. I developed a program source editor, Software Concordance’s central tool, that allows developers to embed multimedia documentation and hyperlinks in their source code, while still supporting advanced program analysis services, including lexing, parsing, and type checking. The environment is designed to integrate all stages of software development life cycle.

    Roles: Research, architecture design, development, testing, presenting in research conferences

    Publication: The Software Concordance: Using a Uniform Document Model to Integrate Program Analysis and Hypermedia. Satish Chandra Gupta, Tien Nhut Nguyen, Ethan V. Munson.
