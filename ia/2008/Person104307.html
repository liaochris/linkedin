Keith Bennett’s Summary

Curious and studious, I love to learn new ways of being more productive (thus my current interest in Ruby and Erlang). I get my kicks writing software that makes the complex simple.

My interests spread wider than software engineering, though, and include world events, international travel, foreign languages, and writing (see blog at http://keithrbennett.wordpress.com). Although I have done well with heads down software development, encountering new people and places would be a welcome enhancement to my work life. I have been thinking seriously about expat life, probably in Asia. Towards that end, I have been studying languages and visiting to make contacts and familiarize myself with life there.

    (Computer Software industry)

  • Senior Software Engineer

    Solutions Made Simple, Inc.

    (Privately Held; 11-50 employees; Computer Software industry)

    2006 — 2008 (2 years)

    The project on which I worked was an innovative commercial Java based ETL product. We built a framework in which pluggable components could be assembled to access diverse data stores (DBMS, files, sockets, etc.); transform the data in various ways; and fan the work out to multiple machines for speedy processing. We provided stock components, and the customers could write their own as well. I was the first developer hired by the architect/lead developer/product manager, and the team and the product's success has grown considerably since.

  • Senior Systems Analyst

    Westat

    (Privately Held; 1001-5000 employees; Research industry)

    1997 — 1999 (2 years)

    Converted DOS-based interviewing system to a web-based CGI executable running on Windows NT. Participated in porting the Wesvar statistical package to WIndows 95/NT.   • Senior Software Engineer

    Hill-Rom

    (Public Company; Computer Software industry)

    1992 — 1997 (5 years)

    Technically, I was a contractor to Hill-Rom through The Registry. (Yes, for five years!) We were three guys in a town house office in Annandale putting together a patient/nurse communication system in C++ under OS/2. This was one of my favorite jobs; Bob Jennings was a great boss, and I learned a lot of new stuff (OS/2, TCP/IP, socket programming, Named Pipes) and had the chance to exercise my object oriented design skills.

    One of my favorite achievements was writing a pluggable networking framework. After writing the product under the assumption that we would only be using named pipes for networking, we received a new requirement to support sockets as well. I abstracted the Named Pipes functionality into a single interface (well, class in C++), and wrote the named pipe implementation from the existing code. Then I wrote a sockets implementation, modifying the superclass as necessary along the way.

  • Software Engineer

    Pacificorp Capital (now defunct)

    (Privately Held; 51-200 employees; Financial Services industry)

    1989 — 1991 (2 years)

    Senior developer on a team writing C programs interfacing with Oracle to manage this finance company's data. Enhanced operations by:

    * writing a function library to simplify Oracle access for the team (previously we had used arcane Oracle function calls with many unused arguments)
    * writing a custom companywide menu system in C, using Vermont Views as the user interface library, keeping menu data in memory for speed, swapping itself to disk when selecting a program.
    * modified menu system to use Paradox Engine for its data store
    * researched use of Objective C as a more productive development language; we were bought out and downsized before we were ready to implement the results of this research.

    It was a pleasure working under Ted Friel and Tom Pfotzer here. They provided intelligent guidance, with just the right amount of involvement -- enough to be helpful, but not too much to limit my freedom to find the optimal solution and implement it.
