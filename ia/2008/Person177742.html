  • Software developer

    LIMSI - CNRS

    (Research industry)

    December 2005 — January 2006 (2 months)

    I had to develop a real time facial animation system based on control points deformations (MPEG-4 decoder & automatic weights computing). Then I linked different modules developed in the lab:
    • A First system, analyzing natural language, which generates dialogs.
    • A second one, receiving data about language, which generates multimodal information in time.
    • A Text to speech system giving use visemes in time.
    • The MPEG-4 decoder transforms those data in control points displacement using mesh parameters defines in a first program.
    • A facial animation system deforming the mesh using weights automatically computed by the same program.

    • See : http://smorelfr.free.fr/Vacation%20-%20LIMSI%202005.pdf

  • Trainee

    VRLAB

    (Research industry)

    September 2003 — July 2004 (11 months)

    I had to animate an avatar playing saxophone in real time.
    • State of the art
    • Skeleton animation
    • Inverse kinematics
    • Smart objects: A 3D Studio max plug-in has been developed to specify behaviours for the object with XML export. Positions, vectors, animations. Then a VHD module could read these data and access to those behaviours.
    • The main principle is to make the fingers follow the object’s parts. In such a way, we have to prepare data from the midi file to know which IK constraint is associated with which key and finger at each time.
    • This is a simple way to have a system sufficiently generic to animate different kind of objects such as trumpet, clarinet …

    • See : http://smorelfr.free.fr/Stage%20VRLAB%202004%20-%20virtual%20saxophonist.pdf
