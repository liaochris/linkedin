Joe Munoz’s Summary

I am a Software Architect and Engineer with substantial experience in the construction and maintenance of content management systems and electronic commerce websites. I am quite passionate about digital media, web-development, the principles of "Web2.0" (a culture of openess, user-generated content, and content aggregation), and programming languages (though I have to admit that Java is my favorite). I have experience in all stages of the software development cycle; be it scope and requirements gathering, high-level system architecture, database schema design, server and client-side component construction, or just sitting down in front of the computer with a pair of headphones and coding until dawn.

Joe Munoz’s Specialties:

Content Management, Messaging, Distribution of Digital Media, Documentation of Software Design and Architecture, XML (DOM, SAX, XSL, DTD, Schema)

  • Senior Software Architect

    Movielink, LLC.

    (Privately Held; 51-200 employees; Computer Software industry)

    January 2005 — September 2007 (2 years 9 months)

    I oversaw the maturity, stability, and consistency of all of Movielink's software systems. I was also responsible for identifying and stewarding best-practices from a design-perspective across the entirety of Movielink's various software applications.

    I also maintained high-level design artifacts including but not limited to domain and data models, as well as, high-level design and/or whitepapers.

    Additional responsibilities included serving as a mentor to the Engineering team and making sure that the developers are aware of various new technologies and software development methodologies that can help to make the team more efficient.

    Finally, I worked closely with various Movielink stakeholders to develop and maintain systems architecture roadmaps, that shrived to ensure that the systems architecture satisfied near-term business objectives while being sensitive to longer-term objectives.

  • Senior Software Engineer

    Sony Pictures

    (Public Company; 10,001 or more employees; SNE; Computer Software industry)

    May 2003 — December 2004 (1 year 8 months)

    At Sony I helped to design and develop a Content Management framework that enabled graphic designers and producers to generate and publish web-content. This framework integrated with a relational database and a content management system to manage digital content and metadata. The technologies used to create the framework included J2EE, XML, XSL. Some presentation elements were implemented using Struts.

    This framework was used to publish the Sony Pictures websites in the US, UK, and Japan as well as for Sony's Connect music download service that was meant to compete with Apple's iTunes Music Store.

    My responsibilities included control over the design of the framework Java API and web application resources (JSP, XSL, Portlets, and XML) as well as development support to the businesses that were using the framework.

  • Senior Software Engineer

    iManage

    (Public Company; 501-1000 employees; Computer Software industry)

    February 2002 — May 2003 (1 year 4 months)

    I served as the department Java developer specializing in customizations for the WorksiteMP J2EE application. My responsibilities included design and development of client projects as well as training the team consultants in Java technologies related to all customization work. This included design and implementation of database schemas, application architectures, custom web applications, Java components, and XSL transformations.

    This position expanded upon what I had learned as a consultant for Cambridge in that I was actively responsible for being a mentor to the internal team, and also had more visibility to external customers.

  • Senior Consultant

    Cambridge Technology Partners

    (Public Company; 501-1000 employees; Computer Software industry)

    August 2000 — December 2002 (2 years 5 months)

    For Cambridge, I was responsible for creating technical documentation during the scope and design phases of a project. Once the project began, I served as software engineer specializing in Java development. While here I specialized in middleware development working with the JMS API, Tibco Rendezvous, and Microsoft Message Queue.

    As a consultant I got the opportunity to expand my documentation and presentation skills and began learning how to express software designs using standard methodologies, UML being chief among them.

  • Associate Software Engineer

    IBM

    (Public Company; 10,001 or more employees; IBM; Computer Software industry)

    August 1999 — August 2000 (1 year 1 month)

    I was a member of the Enterprise Information Portal Team where we were trying to create technology that could organize information tailored to the specific needs and personality of the individual accessing it.

    I helped to develop a "knowledge management" server and client API that was to be part of The IBM Enterprise Information Portal. I was initially responsible for the design and development of the query package of the Java API (constructed in C++ and Java). That API used JDBC and Embedded SQL to retrieve personalized data from a DB2 database.

    Later I designed and developed a Java Trace package that was used throughout the API to log errors and messages.

    I internally published technical reports detailing the architecture of the server and the architecture of the query portion of the API and also helped provide supporting technical documentation that was used in patent applications for the work we were doing.
