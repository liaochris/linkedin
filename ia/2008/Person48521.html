Darren DeRidder’s Summary

I currently work as a software architect with Rialto Media Corporation, a fully owned subsidiary of Alcatel-Lucent, joining as the fourth member of the team to help in forming the new spin-off.

My expertise is primarily in distributed real-time systems for telecommunications software using C/C++ and Solaris/UNIX, but I also have an interest in rapid application development (so-called agile technologies), web applications, and what has come to be called "software as a service".

At Bridgewater Systems I got a bit of a reputation as a visionary and innovator. I introduced a university research project into Bridgewater's AAA software - an invention which allowed new telecommunications usage data record types to be supported without any custom programming effort. It turned out to be very successful.

Later I spent two years on Bridgewater's User Interface team and created a new framework for building highly-customizable and extensible web-based customer and administrative portals for dynamic broadband management. This was a major change for user interface development at the company, and its used today in Alcatel-Lucent's SSC product. It has allowed customers to develop additional features such as user self-registration, as well as re-branding and translation into several languages very quickly (3 days to implement user self-registration, by a customer, with minimal technical services support).

Before leaving BWS I was part of a small team that produced a Policy Server prototype that generated a lot of interest at trade-shows and customers around the world. According to PLM, a key selling point was the user interface, which I designed using newer Web 2.0 technologies.

Darren DeRidder’s Specialties:

I lean towards C/C++ for high-performance telecom software development and towards web technologies for user interfaces. I feel that technologies like LAMP can be used in applications beyond just "dynamic web pages" and provide real value for core network functions. I'm a believer in test-driven development and automated black-box testing, and I continue to dabble in rapid application development technologies like AJAX and XUL. I'd eventually like to start my own company.
