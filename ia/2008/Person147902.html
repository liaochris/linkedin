  • Software Engineer

    Academic Computing Services

    (Educational Institution; 5001-10,000 employees; Information Technology and Services industry)

    October 1996 — May 1999 (2 years 8 months)

    Designed, developed and implemented a full life-cycle rewrite of a computer lab usage system (Check-In 2.x). The application keeps track of computer lab resources. The application used a home grown middle-ware that pinged on serialized C++ STL objects. The server used the mSQL database that would keep track of client side changes.

    Components of project:
    * Client: Windows NT/95/98 application written in MS Visual C++ Developer Studio 6.0.
    * Server: UNIX (Solaris 2.6) in C++ using GCC and mSQL for the database.
    * Proprietary C++/STL based message serialization middleware: this was a message oriented middleware (MOM) that pushed events to clients to keep them all in sync with the current state of the computer lab system. The decision was made to create this in house for lack of funding reasons and because there wasn't something that gave us exactly what was needed for the task.
    * Java 1.1 was used for server side testing.
