Architect

Public Company; 51-200 employees; Broadcast Media industry

August 2006 – January 2007 (6 months)

Peter was hired in to produce a web based Centralised Messaging System, to allow users to enter messages to be sent to selected players, over the satellite link. Peter built the system in a LAMP environment, running under a Unix server, using OO PHP v 5 on the server, and a mix of Javascript and CSS2 on the client side, communicating via Ajax links. A secondary application, run as a Unix cron job, picks up the relevant messages, and produces custom designed XML to be sent to the players over the satellite link. This is currently a ‘fire and forget’ system, so the messages are sent on a regular basis during the dates within which they are valid.
The client side allows for a preview of the screen display, using Flash versions of the screen media, underlayed with the text of the messages.
Peter is also responsible for moving the systems from the currently used Firebird database to PostGre Sql 8.1, using his previously used Object design.
