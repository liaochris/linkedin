John Ouellette’s Experience

  • Co-Founder / Architect

    Rule #6 Enterprises

    (Privately Held; Information Technology and Services industry)

    January 2010 — Present (1 year 2 months)

    In this role I created the vision and architecture for a 3-tier Financial Calculation based web site using Linux, Apache, MySQL and Python. The security architecture was designed to provide a high level of security, using the "OWASP Top 10" as a guide. During the roll out I worked with our developer to ensure we were taking steps to maintain that level. In doing an overall site assessment some of the tasks I performed were:

    ● Hardened the base Apache SSL install ensuring strong SSL/TLS versions and cipher use and then validating against tools such as SSLDigger and SSLScan.
    ● Performed Vulnerability Assessment on the Web Tier using tools such as Nessus, Nikto, Qualys Web Application Scanner ensuring the appropriate risk factors were mitigated, as well ensuring PCI Compliance.
    ● Performed SQL injection assessment scans using tools such as Sqlmap (with and without user authentication) and manual testing. Created shell scripts to automate cross site scripting (XSS) attack checking.
    ● Insured HTTP resource exhaustion and denial of service attempts are thwarted using Mod-Security, Blacklist and Httpd-Guardian, including testing.
    ● Analyzed HTTP session randomness with tools like WebScarab and Burp Suite.
    ● Set requirements for Developer and Systems Administrator access using passphrase-protected Public Key Authentication for remote SSH access. Hardened SSH server configuration with industry best practices (disable root logins, public-key authentication only, disabling password authentication).
    ● Installed host-based Iptables stateful firewall to allow only needed services.

    During my tenure at Google I was also selected to travel to multiple European offices to help out with the foreign work load and train new hires.

    Additionally, a favorite project of mine was scripting a data mining tool for an in-house CRM application to eliminate hours of team members manual work when tracking customer issues.

    Finally, I also wrote a great amount of internal documentation, as most of the procedures that I performed were either newly discovered, or as yet undocumented.
