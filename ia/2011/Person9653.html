Goal: To find a position which offers challenging problems in an environment which values its developers.

I am a talented front-end web development. Not only have I played a major part in several award winning projects as a Flash/Flex developer, I have written articles for notable websites and periodicals in the industry and I have built several libraries for which make Flash development faster and easier..

I have done a lot of work on the layers of communication between the browser, Flash, Flex and the back-end (mostly written in PHP and Java though there is some Python). This has used centralized event dispatching, LocalConnections, and even JavaScript on the front, and it has involved HTMLRequests, RemoteObjects, and XMLSocketConnections.

I'm also very interested in back-end technologies and applications development. The most interesting tasks at my current position involve work in the Java back-end. I've built websites in PHP and I am currently working on a port of the KeepOriel (currently PHP 5, MySQL 5) server to Google Apps (Python 2.[5+]).

KeepOriel is a series of browser plugins (Chrome is in Beta, Firefox is soon to be in Alpha as of 11/11/2010) coupled with a web back-end. The first plugins simply store and restore a spot in a Google Docs PDF through two XMLHttpRequests. The service is also completely RESTful -- a request merely queries a record.

The original PHP/MySQL back-end has been working for over a month (as of the same date), but, since Google Apps will provide limited free hosting, it will be migrated to python. This is also consistent with the eventual desire for the server to be easily installable on any given web server quickly and easily.
