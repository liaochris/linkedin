Rob Bamforth's Experience

IT Manager

Reactiv Media Ltd

Privately Held; Marketing and Advertising industry

March 2008 – Present (3 years 1 month)

I work as an IT Manager, specialising in software development, Java and database administration in MSSQL server 2005 & 2008 and MySQL.
I manage a team of developers, and oversee all aspects of IT for Reactiv Media which has now grown to a size of 70+ employees.

I began developing a bespoke call centre CRM software package in 2007- Customer Prospecting System (CPS). CPS runs on 50 PC’s, 6 days a week. The increased agent numbers and run time meant that I was required to set-up two additional dedicated SQL servers to run the database behind the call centre and the reporting services.

I was also set the task of allowing ‘live-feed’ data to feed into the call centre. Using a combination of MySQL and MSSQL databases this was achieved, with a customer getting a call within seconds of completing an online request.
I also integrated CPS with a dialler package in Match 2010.

Another achievement was a project called iBlade (intelligent Bespoke List and Data Engine). It is a cross-platform software application that fronts multiple databases to create complex database quires. It has brought increased productivity in Production and Customer Service. It also means that people with little to no database knowledge can generate extensive database selections and queries by clicking buttons, selecting fromdrop-down boxes and using drag & drop.
Key parts of the software include; Drag and drop function to select database fields, Include / exclude logic, OR logic, Geographic radios inclusion / exclusion, Data extraction, Save / load count logic, Cross section splits

The application runs using Java Web Start technology meaning that it can run on a remote PC to connect securely to the data server.

Database & Software Developer

Reactiv Media Ltd

Privately Held; Marketing and Advertising industry

May 2007 – March 2008 (11 months)

In May 2007 Reactiv Media Ltd was established.

I was tasked to expand the call centre CRM software application. This included taking the basic package that I had created for Discovery and adding extra functionality such as expanded the level of additional data that could be collected and mapped back to the central SQL database.

I also created a bespoke method of integrating the call centre front end GUI with a PBX telephone system. This involved complex coding algorithms allowing each agent to log into the telephone at their desk to make outbound calls by clicking a button on the software. The CRM now had the ability to; Make outbound calls; Accept incoming calls; Place a current call on hold; Retrieve a held call; End a current call; Transfer a call.
As a result of the PBX integration we also had the functionality to generate complex calling statistics.
A reporting engine was set up to process and auto-generate complex reports for many aspects of the call centre activity.

I also spent time expanding the server and hardware set-up to ensure smooth running of the Production side of the Company. I expanded one SQL server to three dedicated servers. This presented problems in terms of ensuring that all three servers were up-to-date and in-line with each other. I decided to set up SQL server replication which runs to keep the servers up-to-date.

Database & Software Developler

Discovery Media Solutions

Marketing and Advertising industry

March 2007 – May 2007 (3 months)

A position of responsibility in the production department. My role included; Using SQL / T-SQL to query a number of large databases to extract and modify data, Creating new procedures in SQL to reduce time wasted on repeated tasks. This included stored procedures on SQL server 2003 & scheduled jobs. Software development on a bespoke call centre CRM system.

Discovery Media Solutions had a modest sized call centre and I relished the task to create a bespoke CRM software application to run on the agents PC’s. The agents originally used Microsoft Excel and data was supplied to them each morning. I developed a Java GUI application that fronted a central SQL database. This involved created a new ODBC / JDBC connection allowing multiple connections between the agent PC’s and the central database. The agents could now read in records one-by-one, make the call and write the collected data back to the database.

I was also responsible for maintaining the accuracy of the consumer databases held by the company. This included taking data ‘cleansed’ through the call centre activity and matching it back to the database.
