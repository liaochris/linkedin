Multi Theft Auto

Computer Software industry

May 2003 – July 2006 (3 years 3 months)

Architected and implemented the underpinnings of the popular open-source Grand Theft Auto modification. The modification started out performing direct-memory writes from another process to change the values of some variables used by GTA, which ultimately affected player position, car direction and acceleration, etc. The new architecture was highly modularized and shifted the game control strategy into an in-process injected library. This strategy was not new in the world of Windows applications (used by game cheat developers extensively) but was novel in how it was used to extend value of a game.

* Implemented the world's first in-process control mechanism for extending the value of a game (both GTA: Vice City and San Andreas).
* Architected a ground-up framework codenamed "blue" which introduced a modern componentized stack.
* Created support for hooking into the DirectX (versions 8 and 9) which allowed the modification to draw images and text on the screen, in 3D space, further extending value of the game and making it easier to identify players during online play.
* Created a rudimentary modification system (which was ultimately extended by other great developers to support LUA).
* Integrated a voice module using an open-source LGPL library to introduce online voice communication. The code was tested but never shipped.
* Created DirectX graphical libraries to support many new elements introduced by MTA into the game.
* Introduced initial support for patching and trampolining parts of the game to gain fine-grain control. This was later extended by other team members.
