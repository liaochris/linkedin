David Dulak's Summary

I love being challenged and can't stop learning how to improve my projects whatever they may be. My web design and my photography are prime examples of this. If you look back on my body of work, it is very obvious that I am constantly reinventing myself with new found skills. I can't imagine getting stuck in a rut. I have to learn something new!

While I do love to learn and am always looking for great new technologies and techniques, I also have a grasp on what is in good taste and what works from a users perspective. Maybe that's why "I'm a mac." I believe that cutting edge technology is not enough anymore. The user experience is just as important as the technology and I try to live and breath that philosophy. I'm a Computer Engineer with Art and Design skills. I may not be at the level of Apple yet, but that's my goal and my passion.

I use both sides of my brain. I am great at programming and think like and engineer, but I am also very creative and do well at graphic design and photography. I do well as the middle man, because I understand and can talk to both types of people in depth.

I'm proficient in all major operating systems and server platforms. I have been network administrator for OSX servers, Windows Servers, and Linux servers. I've tried them all, but I'm in love with Apple!

David Dulak's Experience

Web Architect

Mutual Mobile

Privately Held; 201-500 employees; Computer Software industry

May 2012 – Present (1 year 5 months)

Contributed on the final sprints of an enterprise iPad app written in web technologies and wrapped in PhoneGap. I mainly worked with the presentation layer and bug fixing.

Led a small team to convert the previously mentioned PhoneGap app into a fully functional Windows 8 app including a few native additions such as live tile notifications and snapview layouts.

Contributed to a R&D project by creating a full javascript MVC web application to plot and manipulate SVG points and paths. The drawings where bound directly to the data in such a way that any change in the data would result in a visual update to the rendered drawing. All SVG drawings saved state allowing undo and redo of any data or drawing manipulation. The connection to the server used server sent events for immediate rendering to the screen when new data arrived from the server. For this project I used the Lavaca framework (getlavaca.com).

Contributed to a Windows 8 demo application written in HTML5, CSS, and WinJS. I helped build a data bound grid view and created the search functionality using the native Windows 8 search contract and displaying the results in a list view.

Contributed to an internal web application to update and track employee resourcing for project managers. This project used Twitter bootstrap and Python with Jinja2 templates.

Currently I'm leading a development team on the Mutual Mobile's first full featured enterprise Windows 8 tablet application for an external client. Funny thing is, I've not been a huge Microsoft fan and have never opened Visual Studio until recently. But Microsoft finally got on board with modern standard web technologies which allows me to jump right in and comfortably build native applications. Yes, I still prefer building open source web applications, but surprising as it sounds, I am enjoying being a pioneer on this new platform. The beta of this application has been submitted to the Windows 8 app store.
