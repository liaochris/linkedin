Germán Río Ibáñez's Summary

I am a web developer, who is very fond of client-server web applications. This is, web applications entirely programmed with javascript that communicate with back-end through REST services.

The front-end is in charge of all related with the interface, this means, layout, components, events, styles,... while the back-end is responsible of business tasks like, validations, data management, user communications,...

Using this strategy, responsibilities are well defined in both sides, and, defining concrete communication contracts between back and front, allows both teams to work separately.
