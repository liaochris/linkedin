Andy Leonard's Summary

I share my passion for data-related technologies with just about anyone. I specialize in data integration - which is a fancy way to describe moving data from one place to another - using Microsoft business intelligence products, especially SQL Server Integration Services (SSIS). I have authored chapters included in seven books, leading the author team for SSIS Design Patterns. I train data integration developers using courseware I developed. I enjoy serving clients as a data integration architect, CSO (Chief Servant Officer) of Linchpin People, and software architect.

Specialties

SQL Server Integration Services, Team Foundation Server, SQL Server database architecture and solutions design, SQL Server performance tuning, SSIS performance tuning, Developer Community leadership, and Application Development.

    • Apress
    • September 5, 2012

    SQL Server 2012 Integration Services Design Patterns is a book of recipes for SQL Server Integration Services (SSIS). Design patterns in the book show how to solve common problems encountered when developing data integration solutions. Because you do not have to build the code from scratch each time, using design patterns improves your efficiency as an SSIS developer. In SSIS Design Patterns, we take you through several of these snippets in detail, providing the technical details of the resolution.
