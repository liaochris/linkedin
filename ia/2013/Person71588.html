Jeroen Pluimers' Experience

Software Development and IT Infrastructure Wizard (.NET, SQL, C#, VB.NET, Windows, XCOM, MFT, TIBCO)

Nationale-Nederlanden

Public Company; 1001-5000 employees; ING; Insurance industry

October 2011 – Present (2 years 3 months) Rotterdam/The Hague/Home

This started as a seemingly straight forward project to migrate the existing .NET 1.x and 2.x based C# and VB.NET software suite (that just had been outsourced after years of hardly any maintenance) running on Windows XP + Office 2003 to Windows 7 + Office 2030 and Visual Studio 2010 keeping the main database backend at SQL Server 2000 on Windows Server 2003 (to future migrate that into SQL Server 2008 R2 or higher) while "keeping the kitchen open" i.e. allowing for new development and corrective changes.

A big precondition: Windows XP + Office 2003 should co-exist for an undetermined time.

Based on a reasonably functional DTAP based environment, it started with some careful planning and dependency checking and the usual impact analysis (coping with UAC, hard coded "Document and Settings" paths, etc).

Soon, new projects raised their head, all with very aggressive time frames:
- Windows 7 would be in a different domain (with a two way trust, all machines in the new domain, all the users in the old onw)
- Support both Office 2003 and Office 2010 on Windows 7
- Migration of the all the servers (SQL Server, Oracle, Imaging, IBM System i - aka iSeries or AS/400 - IBM System z - aka zSeries, System/360, Mainframe or Host) to a new data center with different security rules and (for the Windows servers) a new domain.
- replace the current enterprise bus with Tibco
- Splitting the organization into two functional parts (Insurance and Banking)
- fresh HR system with new person IDs

Because of my history within NN and ING, my roles are now these:
- architect solutions that will survive these migrations with the least amount of effort, soonest delivery dates, and best overall stability
- knowledge source for existing functionality, technical decisions from the past, the software and present NN architectures
- liaison between NN and the outsourcing partner
- key person to keep the testing crew busy
- general problem solver, bug hunter and "guy that gets things done"
