John Madrak's Projects

  • Board of Trustees Portal

    Team Members: John Madrak, Melissa Miller, Jane Gilbert

    Created a PHP solution based on my own custom MVC framework for the Board of Trustees at La Salle University. This system has communities which each individually feature separate private discussion forums, file management (and access control for those files), a simple CMS for content driven pages, a mass email system and an area where you can view the biographies and photos of each committee individually (or the board as a whole). It also features a search which allows you to drill down to specific details to get the results you desire taking into account which communities you have access to. Access to this system is controlled via a custom user account management interface which also gives users the ability to recover their accounts if necessary.

  • Portal Targeted Content System

    Team Members: John Madrak, Alicia Stonesifer, Melissa Miller

    Created a custom content management system to allow for users to submit announcements, flyers, emergency alerts, job postings, photos of the campus and polls. These items can be targeted to specific groups of people in the university by role, major or class year (in the backend, this uses queries into our Banner system) as well as specific time periods. To save resources and bandwidth for our internal web servers, all image based content is uploaded using the Flickr API so that Flickr may be utilized to view the content instead of our own local resources. After submitting, there is an administrative console which allows specified individuals to either approve or deny a submission for display. Once the time period that the submission is meant to be shown is over, the item automatically moves itself to the archive from which the administrator can edit it if necessary.

  • Class Cancellation Notification System

    Team Members: John Madrak, Melissa Miller

    The Class Cancellation Notification System is an application which accomplishes two goals. First, it allowed for our students who had existing e2campus accounts to connect those accounts to their actual usernames so that we could allow them to use CAS authentication for a SSO experience. I accomplished this goal by utilizing a series of API commands which checked for existing accounts and renamed them as necessary or created them when they did not exist. The more important part of this application, however, was then utilizing those same APIs to allow professors at the institution to send out SMS/Email blasts when they had to cancel a class' meeting. The system utilizes our Banner databases to pull out the professor's current courses, the students in those courses and then allows the professor to select the date which they would like to cancel class for. Once confirmed, an SMS message and email are sent out to all of the students registered for that class informing them that the class has been cancelled for that day. The system also has a user access control portion which gives deans (and other individuals) the ability to delegate access to various individuals on a by-school basis based on what they originally had access to as well as cancel classes for the entire school they're responsible for as opposed to only their courses.

  • Course Ticker

    Team Members: John Madrak, Ed Nickerson, Melissa Miller, John Caputo

    The Course Ticker is a PHP application based on my own custom MVC framework that allows authorized individuals to create electronic rooms (public or private) for discussion. After creating a room and starting the ticker, the audience can then join those rooms using their devices (if it's private, they'll need to specify a passcode provided by the presenter), specify a nickname and begin talking to the rest of the audience and the presenter as well. After sending a message, the message will scroll across the presenters screen in a ticker-like fashion. The system automatically gives La Salle University faculty access to create rooms but allows for them to give their associates at other institutions access as well. The actual audience submission portion does not require an account. All of the messages submitted are later logged and stored for easy viewing and download by the presenter if they would like to have a local copy of the submissions as well.

  • Portal Chat

    Team Members: John Madrak

    Created a series of REST APIs that allowed for users who were in our Luminis Portal to communicate with each other over instant messaging. It had features such as "now typing" notifications, statuses, the ability to "go invisible" and various chat notification sounds utilizing jQuery to make the appropriate API calls. I had plans to move towards web sockets, buddy lists, group chats and other enhancements but the project was stopped due to a lack of server resources.

John Madrak's Summary

I am a self-motivated full-stack web application developer (CentOS (preferred)/Debian (capable), Apache, MySQL, PHP, jQuery, HTML, CSS) who is passionate about creating powerful custom web applications and staying up-to-date on the latest trends. Recently, I've begun creating my own MVC Framework in PHP (for the customizability and learning experience) which has been utilized in several of my own custom web applications, most notably the custom Board of Trustees Portal and Course Ticker Application (see project list for details), as well as several other developers' projects due to it's small learning curve and ability to increase rapid (but maintainable) development. Recently, as a side business, I've also been managing a geographically dispersed team of developers and designers on various projects. In this position, I discuss projects with my team daily, oversee their progress and assist with both front-end and back-end development questions as they arise.

In addition to my primary focus as a full-stack developer, I have also had the opportunity to work with PL/SQL while modifying and updating existing packages for La Salle University’s Student Information System as well as some Java/JSP development while working with BlackBoard Building Blocks.

Finally, I have also managed several Proxmox Virtual Environments running both Windows KVM Instances and CentOS Virtual Containers. Most of the time, these CentOS environments were running Apache (or occasionally nGinx), PHP (mod_php or PHP-FPM), MySQL Server, PureFTPD (or more recently, Gitosis) and a set of custom IP Tables Rules while operating as web servers for my employers, clients, or my own web applications.
