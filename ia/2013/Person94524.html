Michael Rivera's Projects

  • Project PAALM

    • May 2013 to Present
    Team Members: Michael Rivera

    The result of an independent research endeavor, Project PAALM is a fast, approximate and inexpensive method for obtaining the phalangeal joint angles of the hand using 3D gesture detection device, the Leap Motion Controller. The framework directly integrates the Leap Motion Controller into 3D modeling software, Autodesk Maya, to create an intuitive user interface for animating hand motions.

    Demonstration Video: https://vimeo.com/75268546

  • Phase Change

    • February 2013 to May 2013
    Team Members: Michael Rivera, Judy Trinh, Dave Sharples, Corey Novich, Kathy Zhou

    As part of a self-organized university course, PhaseChange was conceptualized, designed, and implemented to provide a platform for collaborative audio creation and visualization. The web application allows users to record, layer and shape 30-second sounds in a version control-like fashion. Contributed sounds are visualized as spherical nodes in a sound tree. Each branch of a sound tree denotes a variation in collaboration.

    Anyone can explore and listen to sound trees. Contributors, however, can also track their individual uploads and any associated sound trees. Built using HTML5 (Web Audio API), Javascript (Node, Backbone, JQuery, WebGL, Three.js), the DropBox API, the Facebook API, Socket.io, MongoDB. Team project.

    NOTE: Requires Google Chrome as it is the only browser that fully supports the Web Audio API functionality.

  • Brobots

    • February 2013 to Present
    Team Members: Michael Rivera, Nicholas McGill

    Brobots is a collaborative gesture-based robot control system using Leap Motion Controllers and WebSockets. One user directs the speed and turn angles of a robot while the other user controls the robot's arm and gripper. The two users can be anywhere in the world and control the robot simultaneously. As an additional feature, the robot broadcasts a live video feed to a Google Hangout, making it easy to see how the robot is functioning while allowing its users to communicate effectively. Won 3rd place and a $500 prize at the PennHacks Spring 2013 hardware hackathon. Built using Python, Javascript (Node, JQuery), Socket.io, and the Leap Motion SDK.

  • Social Sign

    • January 2013 to Present
    Team Members: Michael Rivera, Nicholas McGill, Natalie Gravier

    Built at the PennApps Spring 2013 hackathon, Social Sign is a friendly tool for learning sign language! By using the Leap Motion Controller, the BadApples team implemented a rudimentary machine learning algorithm to track and identify American Sign Language from a user's hand gestures. Social Sign visualizes these hand gestures and broadcasts them in a textual and visual representation to other signers in the signing room. In a standard chat room fashion, the interface permits written communication but with the benefit of enhanced learning in mind. It's all about learning a new way to communicate. SocialSign- handing communication to you. Built using Javascript (Node, Three.js, WebGL), Socket.io, and the Leap Motion SDK.

    Mentioned in the Make Blog (February 2013):
    http://blog.makezine.com/2013/02/25/hacking-on-the-frontier-of-gestural-input/
