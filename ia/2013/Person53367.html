  • Ad Manager

    • 2010 to Present
    Team Members: Robert Hoffmann

    This is a project which i wrote on my own time for our company to help distribute advertisements easily and in a generic manner.

    Actually it started out that i was too lazy to change advertisement codes on all our sites (we manage over 20) every time our manager strikes a new deal with an ad agency. So now we just slap a generic tag on all our sites and let the ad-proxy do it's magic, and if we need to add new ad code or change distribution percentages we just do it in the ad-proxy back-office, instead of having to open and change an endless amount of files in all our various sites.

    * Automatically detect ad to distribute (format, site, agency)
    * Target users by region, age, sex, etc
    * Load balance ads based on agency performance
    * 3 Billion+ monthly ads
    * ASP.NET 4
    * ~5% CPU usage on a co-located, dual load-balanced machine

  • HealthMonitoring

    • 2006 to Present
    Team Members: Robert Hoffmann

    Another project i wrote on my own time to help track errors across all the sites we manage. It's now the de facto way of assuring the health of all our websites, and allows us sometimes to catch & patch errors within minutes instead of hours or days.

    It uses HeathMonitoring which is a very useful but not very widely known feature of the .NET framework. It allows us to add just a few lines of setup code to the web.config of our sites and in return we are able to monitor all errors in an aggregated way in our back-office.

    Thanks to it, we are able to track all types of errors: SQL, IIS, .NET Framework, Application, and even client-side Browser & JS errors.

    Back-Office
    * Queriable by date range
    * Errors grouped by Machine Name, Exception Type, Error Message
    * Drill down into errors to see machine information, stack trace, request information, browser type, cookies, & more.

  • HeadJS

    • 2011 to Present
    Team Members: Robert Hoffmann, Igor Ribeiro Lima

    A popular multi-purpose JavaScript library, that exposes 3 core features to Web Developers.

    1) Responsive design in browsers that don't support media queries.
    2) On demand script & resource loading.
    3) Browser feature detections like Modernizr.

  • T4ResX

    • 2012 to Present
    Team Members: Robert Hoffmann

    T4 Text Transformation item template for Visual Studio that makes it easier for developers manage localization files in various projects, including websites, class assemblies, and even expose translations as JavaScript files.

    From there on translations can then either be hooked up to a database and managed via a back-office, or classically as RESX files via another tool i maintain that allows one to transform RESX files to EXCEL files for easy exchange with translators & back.

  • JsonR

    • 2012 to Present
    Team Members: Robert Hoffmann

    A lightweight JavaScript JSON protocol proposal which allows one to cut the size of Ajax requests by up to 100% and thus save on bandwidth.

    Original work is based of a simpler version i came up with at work and that helped us cut (ajax specific) bandwidth usage by around 40%. This version is a complete rethink & rewrite of the original.
