Privately Held; 1-10 employees; Information Technology and Services industry

February 2009 – April 2010 (1 year 3 months)

At Cormine Intelligent Data (CID) I have worked as a Java developer on a system called Zoogma. Zoogma is a type of data mining program and can ingest millions of documents either directly or by scanning through desired websites on a schedule. It then runs these documents through numerous analyzers to record information about the document. This way it can provide a way to easily search for the information desired. I have worked directly on numerous components in Zoogma including getting the desired information into the system, the ability to auto-classify the documents into taxonomies, extracting out key terms from a document, handling entities (person, place, location, or date) in a given text, and inserting both the original and derived information into a back-end Solr index to be queried and faceted on.
• Development using Java and some PHP, Python, Ruby on Rails, and Bash.
• Usage and integration of the Spring Framework for autowiring projects together, authentication, SOAP webservices, and database access.
• Using Jersey and JAXB to connect to RESTful webservices.

• Unit testing all code to within 95% branch coverage using JUnit along with EasyMock and later Mockito for mocking out interfaces.
• Installation, use, and management of continuous integration tools such as Maven, Hudson (a continuous build tool), and Nexus (a Maven repository).
• Software version control and management using Subversion, Git, and ViewVC.
• Usage of Eclipse with plugins for Subversion, Groovy, PHP, and Ruby on Rails.
• Software logging using SLF4J.
