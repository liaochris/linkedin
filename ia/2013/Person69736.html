UI Architect

Standard & Poor's

Public Company; 5001-10,000 employees; MHP; Capital Markets industry

August 2011 – Present (2 years 2 months) New York, New York

Retained by S&P's Ratings area to lead and architect the UI for phase 1 of their "RAMP" project. RAMP is a tool used by the analysts to aid in the ratings workflow.

The project was agile, with daily scrums and bi weekly sprints. I joined right at the end of feasibility and day 1 needed to deliver a UI architecture built around ExtJS (a javascript engine that had already been chosen).

Based on my recommendations the project went with a very thin JSP layer, instead 99% of the UI logic is driven by JavaScript that accesses a JSON/API driven by Java middle and Oracle backend. This was particularly important as there was a desire in the future to support ios devices either with custom UI's or native apps.

I then lead a team consisting mainly of novice offshore UI developers, to deliver the UI in incredibly tight deadlines. Phase 1 was 6 months long. Went from nothing, to end to end workflow that will be used extensively every day by analysts. Currently working on release 3 of the project.

We used version 4.0 of ExtJS a combination of OOTB and completely custom UI components:

- complex resizable panel layouts
- trees with custom UI's that appeared disappear
- Grids with custom cell UI's, custom search filters
- Popover Windows that embedded other systems UI's
- Drag and Drop reordering of panels
- Dynamic add/remove for panel stacks
- Panel stacks that could be opened to display lazy loaded custom UI's
- Integration with other existing systems using iframes, and all the associated issues with that approach.

Some key customizations I delivered:

- anonymous pub/sub system for extjs
- integration with ckeditor and spellcheck
- JSON parser and schema validation
- Google Chrome Frame COM object proof of concept in vignette
