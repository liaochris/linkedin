  • Distributed Profiling & Performance Monitoring Tool

    • June 2013 to Present
    Team Members: Jill Burrows

    This is a program that keeps track of running processes, their stats, and their open network connections (though it could be used to track any kind of file descriptor). I created this with the goal of visualizing the interactions of multiple components in a distributed system.

    I wrote this draft in a week during my free time. It is implemented in Ruby, uses immutable data structures, depends on ProcFS being present, and uses a polling architecture. The data structures are mildly patterned after Merkle trees in order to have a hash for individual system states and to allow detecting repeat states.

    If I were going to create a production ready version of this, I would use C or C++, ditch the polling, bypass ProcFS and go straight to the kernel for monitoring changes. This would allow creating different modules for different types of operating systems instead of tying it to systems with ProcFS.

    Allows cellphones to create an ad-hoc adaptive mesh network with routing over wi-fi and bluetooth.

  • StatisticsJS

    Team Members: Jill Burrows

    Embarrassingly basic, yet flexible statistics library focused around finding the minimum, maximum, mean, variance, or deviation of a set. It also allows binning a series into a set of ranges and applying a statistical function to those ranges.
