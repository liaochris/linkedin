Jim Nutt's Summary

I'm a long time software developer (30+ years) who has worked with very nearly every procedural programming under the sun (I've not done anything with functional programming, but I'm working to remedy that). I started on the TRS-80 Model and have worked on everything from DEC-10s to 8051s and pretty much everything in between. I've done machine control, instrumentation, web backends and frontends, security and probably other things that I can't remember at the moment.

Specialties: C, C++, Java, Web Services, Web Services Security, .Net, C#, Automation, Linux, PHP

Node Operator

Fidonet

Nonprofit; 1001-5000 employees; Computer Networking industry

1983 – 1991 (8 years) Phoenix, AZ

I operated at least two different Fidonet nodes during this period. I assisted in developing standards for interoperability between Usenet and Fidonet. I am also the author of the MsgEd editor used for offline messaging by Fidonet node operators. Written in C, it supported Fidonet echo groups (similar to Usenet newsgroups) and standard email. The actual text editor portion of the software was licensed to SEA for use in their Fidonet systems. The source was designed to be as portable as possible between the variety of not quite standardized C compilers available at the time and was released into the public domain in 1990 (the code is available at http://nuttz.org/msgedsrc.zip). I also acted as the interim editor of Fidonews (the Fidonet newsletter) for a brief period.
