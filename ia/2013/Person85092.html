  • SASS integration in Tuenti web platform

    • July 2013 to Present
    Team Members: Marcio Barrios

    I pushed for this project in Tuenti for ages, finally I could change our daily UI creation workflow adding a new layer of abstraction with a CSS preprocessor, in this case SASS.

    Usually this shouldn't be a big project, but imagine to implant SASS in a huge code base with lots of automatic processes, with some dozens of static CSS files and lots of small CSS components through YUI modules loading asynchronously, in a team of more than 150 engineers, and in a platform with millions of users. The pull request with the SASS integration branch was huge and with lots of dangers and manual merges, and I needed to change some parts of our UI architecture to fit the preprocessor properly.

    I had help of some other engineers to make it work with our build process and to be a change as invisible as possible, for example UI developers don't see any CSS file in the repository, only the SCSS files and it compiles automagically in our test environment without the need of running any command in the terminal, so their workflow is almost the same but with some new useful tools like mixins, variables or nested selectors.

    I created with the help of other UI developers a set of base components, helpers, variables and mixins to facilitate our daily work, now every developer in Tuenti can create UI stuff easier, faster and with more consistency. For example they don't need to deal with vendor prefixes anymore, they can create cross-browser gradients with only 2 parameters or use a set of variables for colors and measures to maintain the coherence across Tuenti. I also wrote a SASS coding style guide to set some rules and promote good practices related with SASS.
