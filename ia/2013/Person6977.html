Privately Held; 501-1000 employees; Photography industry

August 2012 – Present (1 year 1 month) Paddington

Recently moved to the I&A team as part of a incubator within Photobox in the Southwark office.

Paddington:
Implemented and executed the following:

* Caching - rewrote parts of frontend caching layer written in perl, which used in-proc, memcached, and file based cache. This involved analysing performance of the application stack and producing reports, testing performance and some primitive load testing. Removing file based cache reduced the load on the servers by half.
* Due to using file based caching and local memcached servers, the templates that were generated were inconsistent on every server. The invalidation of this caching layer previously was largely error prone, and by changing the above is now more reliable.
* Wrote a tool in PHP to compare the hashes of all servers in the frontend cluster to demonstrate the inconsistencies visually.

* Monitoring
* Implemented monitoring with graphite, which received at peak 1.8 million metrics a second.
* This consisted of haproxy, 3 carbon cache servers, 2 carbon relays, 1 web / memcached, which was ran in EC2 with provisioned IOPS.

* Implemented Collectd to graphite and plugins
* Wrote business metrics that sourced data from numerous queries from clustrix and wrote them to graphite
* Wrote clustrix monitoring to graphite (https://github.com/ajohnstone/clustrix-graphite)

* Alerting/notifications
* Setup icinga with mod_gearman monitoring around 100 servers and covered 1200 checks
* Configured various alerts from production servers, internal infrastructure, and a range of application and system level checks.
