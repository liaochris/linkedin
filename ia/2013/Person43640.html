Eric Windisch's Publications

  • OpenStack Security Guide

    • OpenStack Foundation
    • July 1, 2013
    Authors: Eric Windisch, Keith Basil, Bryan Payne, Robert Clark, Gerald Bunch, Malini Bhandaru, Gregg Tally, Eric Lopez, Shawn Wells, Ben de Bont, Nathanael Burton

    The OpenStack Security Guide provides an overview of security best practices, guidelines, and recommendations for increasing the security of an OpenStack deployment. 

    Objectives of the book include:

    - Identify the security domains in OpenStack
    - Provide guidance to secure your OpenStack deployment
    - Highlight security concerns and potential mitigations in present day OpenStack
    - Discuss upcoming security features
    - To provide a community driven facility for knowledge capture and dissemination

    Each OpenStack deployment embraces a wide variety of technologies, spanning Linux distributions, database systems, messaging queues, OpenStack components themselves, access control policies, logging services, security monitoring tools, and much more. It should come as no surprise that the security issues involved are equally diverse, and their in-depth analysis would require several guides. We strive to find a balance, providing enough context to understand OpenStack security issues and their handling, and provide external references for further information. The guide could be read from start to finish or sampled as necessary like a reference.

Eric Windisch's Summary

In addition to those positions and skills included in my resume, I have also been involved with open source development.

Several notable contributions include:
* OpenStack contributor (active 2011-present)
* Porting C/C++ code from the X86 CPU architecture to PowerPC, Sparc, and MIPS cpu architectures on Linux, Solaris, and Irix operating systems. Notable projects include:
- Quakeforge, a video game engine.
- Open Media System, an MPEG2 decoder and user frontend.
* GNU GCC
- Working closely with GCC core developers, designed a test to reliably reproduce a 15 year-old fault in the compiler's optimization routines. Resolution of this bug accounted for performance improvements for software compiled with GCC for big-endian targets.
* Cairo Graphics Library
- Contributed libsdl and SVG rendering demonstration/example code. Several applications, including the popular children's graphics software TuxPaint, have based their SVG rendering on this code.
* SD2XC, a cursor conversion script
- This script written in 2003 was the first to recognize and resolve a need for importing mouse cursors designed for Microsoft Windows into Linux/Unix systems. This script has since become a common, if not essential, tool in the workflow of building and converting mouse cursors for use on Linux/Unix systems. This script has been featured by Nicholas Petreley in LinuxWorld Magazine, his book Linux Desktop Hacks, and has been included in several Linux distributions.
* Fonomo Video Conferencing Plugin for Pidgin
* Cloud::Framework - Framework for building cloud computing infrastructure services. Released as open source in early 2008.
* restful-zfs - a Python solution for providing RESTful access to ZFS and iSCSI management for Solaris.
* pam-dbauth - a highly configurable authentication module for Unix systems with support for several SQL and "NoSQL" databases.
