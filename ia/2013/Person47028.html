Gary Rowe's Summary

Over the last ten years I've worked in financial software development in the roles of developer (C, Cobol and 8086), systems analyst, solution architect (OOAD, UML, C# / .Net), project manager and senior technical writer, and I also created and managed the new release engineering team for a java-based banking system, handling the source control, configuration management, issue tracking and s/w releases across the teams in India, UK and Eire.

I helped hire and train the initial development teams in Bangalore, India for the new Development Centre, set up the development environment and procedures, and then worked there for over 7 years managing and mentoring the teams, processes and products.

On returning to Ireland I acheived 1st class honours at the IMI studying Core Management Skills (Hetac7, 2010) and took the 'Train the Trainer' (Fetac6, 2011) training.

I am currently working as an IT Consultant with CubeMatch, and relishing the new challenges!

Specialties

IT Development, Systems Architecture, Application & Systems Design, Project Management, Solution Architecture, Technical Writing, Photography

Gary Rowe's Experience

Architect / Project Manager

Misys IBS

Privately Held; 5001-10,000 employees; MUSJF; Computer Software industry

February 1980 – January 2008 (28 years)

I worked with systems written in low-level assembler and C, from the pre-runner of Bankmaster to being instrumental in bringing Bankmaster to DOS, then across to Unix, and then the Unix code back to Windows. I specialised in writing and maintaining the common routines for the Bankmaster System, and designing and creating many of the external systems - the File Maintenance system, Menu & Security systems, Passbook Printing, and ATM & POS systems.

I was also heavily involved in many of the projects to advance the spread of Bankmaster LAN to new platforms, from putting a Windows GUI in front of the interactive DOS programs, writing the File Server process on Unix to allow BMLan to have its database on Unix, creating the PC/TCP VDD interface for BMLan for Windows NT 4, to modifying the BM systems code to allow it to use a Relational Database instead of a flat-file system, without changing any applications code.

Lastly I worked in India for 7 yrs, 1st leading the
BMLan Systems development team and then Project Management, running technical projects often managing outsourced development teams alongside my internal ones. I spent a year as a Solution Architect for a new object-oriented Banking system, and then I set up and ran the Release Engineering team for the new flagship java product, Universal Banking, controlling the Configuration Management and Release Control of the java develeopment teams across Bangalore and Dublin.

I returned to Ireland in July '07 as Senior Technical Writer for the UB project in the Dublin office, where I helped set up the current Technical Writing team.

I then left Misys, and have since worked on contract to them twice.
