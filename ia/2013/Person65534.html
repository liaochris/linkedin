Joel Oughton's Projects

  • Improved Network Map Display

    • March 2011 to October 2011
    Team Members: Joel Oughton, Richard Nelson

    Advances in web browser technologies have allowed for highly interactive, cross platform applications that have not been made full use of in the computer network industry. The aim of this project has been to use these technologies to produce an improved implementation of a network map that effectively communicates topology and performance data to network engineers. The development of such map has involved assessing HTML5 technologies, discovering what network engineers value in a network map, researching relevant information visualisation theory and developing working prototypes. Representing computer networks in particular can get difficult when the number of nodes and links is large. Visualisation techniques have been identified and made use of that are effective in addressing the scalability of displays, such as semantic zooming and layout grouping. This along with an expressive way of displaying network bandwidth has made for improved presentation of network map displays that have received good initial feedback in preliminary evaluations.

  • Improved Data Presentation for AMP

    • November 2010 to February 2011
    Team Members: Joel Oughton, Brad Cowie

    The purpose of this project was to continue work started last summer on improving data visualisations within AMP (Active Measurement Project). The work was broken down into three specific areas. The first of which was the enhancement of the dynamic line graph displays to incorporate time series events, improve time navigational features and allow for standalone deployment in other systems. I created a time series event system that allows graphical indicators to be overlayed on the graphs where in time it occurs. More information about the event is found by hovering over it. The time navigation features lacked quick and precise movement. To combat this, I implemented the ability to snap to useful time periods (hour, day, week etc) and then move back and forwards in time based on this period. It is useful for this graphing software to be used in systems other than AMP so I wrote the software to allow for various data input options. The second area was to redevelop the existing static traceroute displays to improve the layout as well as allow for more path information to be included.
