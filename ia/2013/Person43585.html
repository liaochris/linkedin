Ricky Cormier's Summary

I am an expert in cross-platform ANSI C/C++ development, producing high-performance/low-resource solutions using open technologies. I have experience in a large number of the Boost libraries and am a capable meta-template programmer. I firmly believe that good programmers code but great programmers re-use.

Being experienced in writing code for both Windows and Linux, I have an advantage over most developers who may only know and specialise for one platform. I am able to produce a single codebase that will work cross-platform, saving both time and money on products that have such a demand. Meanwhile, I know both platforms well enough to be able to write specialist code as required

As well as being a strong C++ developer, I am also very capable at developing using a number of open-standard languages, such as Perl, Python, Lua and a variety of others. I possess a range of system administration skills for both Windows and Linux and have worked with many flavours of each, both as desktop and enterprise setups.

Working alone, in a team or as a team leader; I am a highly motivated and dependable individual who is able to deliver high quality results within budget to realistic deadlines. I seek a role with a forward thinking technology company that can offer a challenging but rewarding career in the field of software engineering.

Senior Software Engineer

Last.fm

Public Company; 51-200 employees; Internet industry

2009 – 2012 (3 years) London

Responsible for developing and maintaining the critical data services that power the Last FM website. During my time there I was instrumental in adopting TDD as a standard process and phasing out SVN in favour of a workflow that uses Git, Gerrit & Jenkins to provide robust quality assurance during development.

Lead developer for the proprietary database service used for storing all user "scrobbles". The database is structured to be specific for the needs and demands of the Last.fm website, which performs many thousands of queries per second. Whilst working on this project I enhanced the proprietary query language, fixed a number of defects and improved a number of key areas.

I developed a newsfeed service using Thrift as the primary RPC API. The service is a datastore for the main website, displays a feed to a user based upon their favourite artists and friends. For each user the service maintains a collection of circular arrays, each having a different interest priority. The store can be queried in multiple ways to find items of interest to a user based upon the priority filters they are using.

I rewrote the audio fingerprint Service (afps) and its proxy server. The afps proxy offers a HTTP interface to the general public to allow them to query or submit fingerprints for audio files using an open source fingerprint client developed by Last.fm. The proxy was written in java and used Tomcat as the HTTP server. The proxy then communicates with the main service using a proprietary binary format.

Comment spam is a huge problem on the Last FM website. Given my past experience, I was the logical choice when last.fm needed to combat their growing comment spam. I developed a Naive Bayes classifier service using sklearn (python) and libsvm (C++). Poignantly, it was deployed into production on my last day at Last FM.

Senior Software Engineer

McAfee

Public Company; 5001-10,000 employees; MFE; Computer & Network Security industry

2004 – 2008 (4 years) Aylesbury

Co-architect and technical lead in a team of engineers, working on the design and development of a next generation anti-spam engine. The engine was written in C++ and utilizes many technologies including STL, MPL, Boost, PCRE, LibCurl, Mime++ and Lua. Designed to be completely cross platform (initially developed on Linux, Windows and Mac OSX platforms), the engine is to be utilised in McAfee’s enterprise level gateway technologies and their consumer desktop products.

Chief architect and sole developer for a web based collaborative multi-user Spam artefact review system to allow e-mail artefacts to be classified as ham, spam or phish. The solution employs various techniques to automate, as much as possible, the classification process to minimize human involvement. The review system consists of two main parts; back office agents and the main review system. The ‘artefact agents’, written in Perl, trawl the McAfee spam traps looking for artefacts, which could be used to classify e-mail. Where automatic classification wasn’t possible the agents submit the artefact to the review system for manual review.The front end is written in C# using ASP.Net, ADO.Net and XML/SOAP web service technologies.

Designed and developed a ‘Customer Technology Preview’ Browser Help Object (BHO) plug-in for Internet Explorer to protect users against phishing scams. The BHO, developed in C++ using the IWebBrowser2 COM interface, used streaming rules to determine if web sites were potentially dangerous and redirected users to a ‘safe’ landing page allowing them to ‘click-through’ to the destination if required. The code base was eventually integrated into McAfee’s consumer based desktop security products before being eventually replaced by McAfee SiteAdvisor. For this work I received ‘Developer of the Quarter’ recognition and was awarded a one off monetary bonus.
