Education Developer

Ubuntu Linux - Canonical Ltd.

2008 – 2011 (3 years)

Our primary aim was to get Ubuntu to become a platform that would be accepted as the defacto standard for Education, both on the server and on the desktop. On the server, Edubuntu had a real advantage over other operating systems in that it could handle running older hardware through a terminal based system called the Linux terminal Server Project. With this software it was possible to control not just classrooms, but whole schools by running the system in a centralised fashion whereby users would find it very hard to break any individual system, seeing as the server was the only computer feeding images to the terminals which were then being used by the students. It also eliminated the need for buying newer terminals, as the servier itself was responsible for the speed and memory used for each user. In other words, one would gradually increase the specifications of the server as and when it was needed, but the termnald would always run at the speed of the server. Apart from that, users were able to sit at any terminal in the school, type in their user and password and recieve their desktop wherever they chose to sit. It was even possible to create groups of users so that if the science curriculum required a more complex setup with video and macromedia presentations, this would be possible by creating such an environent on the server and then adding the users who would part take

In recent years LTSP has even been devloped to use wireless transmission of images, though there is still no standard as to how this will work, or if indeed the systems are ready to manage that large amount of data streaming through the air without some control mechanism like a mesh.

I also wrote a large part of the LTSP documentation and the official manual, as well as heling to manage the website where possible and integrate sugar, the OS where possible too.
