Andreas Holstenson's Projects

  • High Availability Messaging

    Team Members: Andreas Holstenson

    Developed a solution to enable several disjoint systems to transfer data to systems running at different physical locations. The aim was to enable almost effortless data replication and maintain a high availability for all services.

    * Messaging solution for distributing data to both off-site locations and customer servers
    * Used to keep customer servers up to date with latest aeronautical data
    * Data is distributed to consumers as it arrives from producers
    * Supports master/slave and master/master replication
    * Core servers run in a cluster configuration, using majority voting to coordinate actions
    * Supports redundant producers, when one fails another takes over
    * Guaranteed delivery of messages

  • System platform and application container

    Team Members: Andreas Holstenson

    Developed a platform and libraries that were used to enable easy development of client/server applications.

    * Client/Server architecture with custom network protocol
    * Module based, systems tie together required modules at runtime
    * Support for pluggable storage backends
    * Android support
    * Ability to move modules seamlessly between servers
    * Distributed events used for realtime data updates
    * Clients built in an event based manner, allowing them to always be in sync with the latest safety critical data.

  • Tinder

    • December 2011 to Present
    Team Members: Andreas Holstenson

    Tinder is a knowledge and collaboration platform. It combines discussions with colleagues with sharing content such as documents. The entire system is cloud based making it easy to get started.

    On the technical side Tinder is a modern system with a foundation written in HTML5 and JavaScript with WebSockets for most client-server communication.

    * HTML5, JavaScript, CSS, Java and C++
    * Frontend built back to back with backend to enhance the user experience
    * Real-time updates for web clients using Web Sockets, streaming and long-polling
    * Lock-free synchronization to support real-time document editing (similar to Google Docs and Etherpad)
    * Full text search engine with support for many different document types
    * Rest API built as integral part of backend
    * Responsive Design
