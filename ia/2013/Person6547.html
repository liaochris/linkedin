  • Rexam VMI (Stock Flow Control)

    • 2005 to Present
    Team Members: Andy Strohkirch, Novelis Deutschland GmbH

    Rexam VMI is an internet based web application which helps to control and organize the flow of goods at several stock locations on a group-wide level. The goods stocked are used by can production facilities and the production locations need to be able to produce products continuously, while the amount of stocked goods shall be kept at minimum. The flow of goods depends on the placement of orders, receiving deliveries and the consumption, which all happens on different locations.

    The application provides logins for different user groups such as Viewer, Forecast Manager, Consumption Manager, Goods Manager and Administrator.

    The Viewer has access to documents, but is not allowed to enter any data. The Forecast Manager is responsible for setting forecasts; the Consumption Manager enters consumption, deliveries and inventory. Occasionally he corrects forecasts and handles returns. The Goods Manager plans and enters deliveries. He is able to define the product list and can open document sheets for data entry. Last, the Administrator is responsible for setting up users and the data structure. He is able to change categories, products and document data.

  • Barcode Manager

    • 1997 to Present
    Team Members: Andy Strohkirch, DHL Global Forwarding GmbH, MSAS Cargo International GmbH, Exel GmbH, Benntrans Verkehrslogistik GmbH & Co. KG, Media Networks GmbH

    Real time software receives jobs from a database application, barcode data from barcode scanners and signals from a set of light barriers. Multiple modes of operation depending on used equipment.

    Per example, a customer can preload a hand scanner with a packing list and scan packages offline. The device will inform the warehouseman if the scanned package is in the list, and advise what to do with the package. Once the user finishes the job, the data can be transferred to the software, which will then process the scanned data and forward it to the database applcation.

    In another example, the warehouseman can preload the software with a packing list and use online scanners, which are transferring the data in real time. Barcode Manager will examine each scan in real time and give out directions to the scanner and/or the monitor.

    Additional the responding action of the warehouseman can be monitored by light barriers, which transmit a signal to the software. The action of the user can be confirmed, or the user can be warned in case he made a mistake (quality control).

    The software can be freely configured by the user. The barcode type, overall length, the start-, end- and stop bit and other structural demands can be adjusted. The light barriers typically are used in sorting streets and the direction and commands can be changed. Possible job types are admission into warehouse, retrieval output, inventory and sorting processes.

    Environment
    · Microsoft Visual Basic v4 / v6

    Team Size
    · 2 plus customer

    · Design of the solution architecture and the interfaces between the components
