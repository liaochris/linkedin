  • .bat Scripting

    • June 2011 to Present
    Team Members: Jessica Noland

    Batch files are an interesting monster. They can be brilliant tools but they can also be destructive. During the summer of 2011 one of my IST instructors asked me to develop a process that could shut down a computer running Windows 7 and the only thing that should come up during start up is an instance of Microsoft Word. At first, I was a bit baffled, trying to think of a way this could actually be accomplished. After doing some googling, I mean researching, I noticed there were many programs that could be purchased that may or may not achieve the desired goal. Counting those out pretty quickly, I found my way into batch files and registry edits. Once I understood that a single registry key controlled the window manager, aka explorer.exe, I realized that this may be my golden ticket. From here I developed a .bat file that reads itself as a registry file. So it runs and processes as a .bat file but utilizes the registry key changes within itself to change the correlating registry key within the registry. It's not a very difficult file to understand, basically what happens is the registry key that normally launches explorer.exe at start up would be changed to launch winword.exe instead. The only snag was that depending on if the client was running the same bit level or not between the operating system and Microsoft Office, the winword.exe file was stored in a different location. Now, for my own curiosity's sake, I looked into a way to disable Task Manager as well because even though explorer.exe was no longer launching during start up Task Manager, which is a separate process, was still available to the user. This is where it could have been dangerous. By disabling Task Manager the only hope left for the user to go back to before the registry edits was to boot into Safe Mode with Command Prompt and run the fix .bat file from there. I also have looked into how to disable Command Prompt with registry edits but I haven't tried that one just yet.
