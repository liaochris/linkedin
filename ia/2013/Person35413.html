John Dideriksen's Summary

Software developer, architect and Project manager for a wide range of embedded system applications. Particular interested in SW design patterns, architecture patterns, idioms, best practices and usability.

Embedded systems and framework programming is my main interest. As the challenges here are some how harder and more fun, than large systems where there's always enough system resources available.

I'm interested in the systems requirements as well as it's usability. My focus is always on the end users and their experience with the system.Where end users are both the actual user of the final product, and the developers maintaining the system internally.

I aim to use the best, free, tools available to create the best user experience for the particular embedded system application.

Specialties

Design: OOA, OOD, Design and Architectural patterns

Programming languages: Java, Ant, XML, C, C++, perl, PHP, Make

Assembler: 80353, x86, RICS 3000, ARM

Documentation: SGML, HTML, UML, the Gimp and wiki

Tools: Emacs, grep, bash, findutils, Apache, php, lint, GNU C compiler and linker, ESIM, Netbeans, Rational Rose, MS office, Visio, and finally firefox and google

System knowledge: Java ME, Mobile phone, Embedded, PC, UNIX, Mac, Linux. Including debugging sessions on these systems
