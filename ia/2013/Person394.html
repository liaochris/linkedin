Contributor

Open Source

August 2011 – Present (2 years)

I am an active member of the LLVM and clang communities, and have commit access to both projects. I have over 50 patches accepted on the trunk.

Several patches have been accepted into LLVM for improved Windows support. These changes include improved threading (r140011, r141907), 64-bit stack walking (r140906), crash recovery (r138199), as well as others.

I have also had clang patches accepted for code generator fixes (r148243, r148359), improved platform support such as corrected line endings when writing out preprocessor files (r140435), improved C++11 standards compliance (r149664, r156271, r157784, r160490, r161088), and support for MSVC's thiscall calling convention (r151122, r151123). Improved the parser's ability to parse balanced delimiters (r141782) for cleaner parsing code, more consistent error reporting, and gracefully preventing stack overflows. My largest contribution was refactoring the way __declspec attributes are parsed and adding better semantic analysis for them (r158717).

I intend to continue contributing to these important community efforts as time permits.

Aaron Ballman's Publications

  • Ramblings on REALbasic

    • Softcover Book
    • October 13, 2009
    Authors: Aaron Ballman

    For the past five years, Aaron Ballman has consistently blogged about REALbasic programming and a host of other subjects. This book is an edited and updated compilation of over 200 of the best blog entries, including new content that's never been seen before. This book contains an enormous amount of information regarding programming in REALbasic. In it you will find tips & tricks for REALbasic that are not available in any other source. Considering that Aaron was one of the senior engineers employed by REAL Software at the time this content was written, this book is a must have resource for anyone serious about programming with the REALbasic.

  • Targeting Windows (too) for Your REALbasic Apps

    • O'Reilly's
    • March 25, 2005
    Authors: Aaron Ballman

    This article is designed to help developers, whose primary experience is with Macintosh or Linux, to create applications to be deployed on Windows. It assumes that you have some experience using Windows, but little experience deploying applications there. This piece isn't a primer for learning how to program, nor does it cover how to write your programs. Instead, it discusses things like user interface differences, usability, general user expectations, and best practices for programming applications for Windows.
