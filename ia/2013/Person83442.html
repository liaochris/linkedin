Reporting and Business Intelligence

The goal of the RBI project is to provide new or re-linked reports and MS Access Database applications (known as Operational Applications) that will replace those which rely on a System of Record (SOR) which is being replaced by a new SOR.

My job as a BA is to work closely with the consumers of the current SOR and make sure that all of the data elements they need to run their current reports and applications so that they can be re-linked and available for use when the new system goes live.

The Developer portion of my role has me performing the re-linking of the Operational Applications as well as writing new SQL Server Reporting Services (SSRS) reports targeting the new SOR.

Energy Components

As a BA I am monitoring and updating the development status of the Interfaces which currently retrieve or provide data to the current SOR and will need to do the same for EC.

This requires communicating with the developer(s) who are making the changes for each interface and maintaining a list showing the current status and any other crucial information related to the interfaces.

With CabbageStash you create a Stash for each child (or you can have multiple per child for say a savings account) which keeps track of their Cabbage (points) balance.
