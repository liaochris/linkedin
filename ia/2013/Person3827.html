Systems analyst / Developer

Guestnet

March 2005 – February 2007 (2 years) São Paulo Area, Brazil

Guestnet was an area of G&S Tecnologia created to supply networking services to hotels market. Like Unicon, this area became a independent company.

Once the systems of Unicon was working fine, i was invited to improve hotels' servers that provided internet access to guests.

As developer, the chalenge was make the internet acces to guests as simple as possible.

To solve problems like fixed ip address, company internal proxy, wrong dns configurations or insufficient user privilege to release and renew DHCP, we developed an integrated system that analyses network traffic and creates profiles to act like the original network configuration. This increased the amount received by internet paid to Hotels and ensured the fidelity of owners of notebooks with "problematic network configurations".

The paid internet could be provided by captive portals integrated with hotels' registration/billing system or by scratch cards with passwords. Each card had a amount of time to internet access.

To provide this service to a large amount of hotels, some tools was developed:
- Basic Linux Box (automatically installed by cd and when online, connect to VPN to be available to setup);
- VPN/Configuration Server (used to setup the server, provide guest information to help-desk and monitor services);
- Version Control of Software (stores each version of software deployed on Linux Box);
- Billing Server (used to register the use of scratch cards and authenticated subscribers).

Systems Analyst

Unicon Tecnologia

February 2003 – March 2005 (2 years 2 months) São Paulo Area, Brazil

Unicon was a departament of G&S Tecnologia which became profitable enough to turn into a company. As a systems analyst that could help with this new moment of Unicon, i was selected to handle internet installations. The challenge was integrate the ADSL Lan deployed in residence condominiums, the help-desk and the payment system. To make a short story, was developed a system that handle clients' informations and payment status. This status was used to generate firewall rules in linux servers. This means that if a internet subscriber had some debit open, a page with help-desk's phone number was displayed on browser. Every interaction with subscribers was stored in this system. This was very useful to keep a history and centralize communication.
