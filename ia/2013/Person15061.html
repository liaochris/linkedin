Fran Hoey's Projects

  • Oxfam Brochureware and Ecommerce Websites

    • March 2011 to Present
    Team Members: Fran Hoey, Chris Gaskell

    Working with Code Computerlove for 15 months as a member of their Sitecore team we implemented an enterprise solution for Oxfam. The solution involved building five different sites within the same Sitecore application and handled their commerce, mobile, brochure ware and community presence.

    Within Sitecore the application is built using custom content components. These components can be created, edited and removed using the page editor or the more standard content editor. This allows the editor to have complete control over content and reduces development time as individual templates for each page type do not need to be created. The Sitecore pipelines where extended to allow management of a custom linking database and application cache.

    The hardware was implemented using two load balanced content delivery boxes and one content author. These servers live in an app-tier behind two load balanced IIS ARR (proxy) servers in a web-tier. This allowed us to offload SSL from the application, cache assets, manage redirects and write custom headers without having to bulk out the application.

    Using continuous integration and NANT scripts it was possible to add the code base to a blank copy of Sitecore at every build allowing Sitecore to be upgraded very easily. The application is built using dependency injection and extensive unit testing.

    The application has nine points of integration including Adobe search, Telligent and a number of bespoke WCF endpoints. By using dependency injection and an object orientated approach we were able to alter the dependencies by environment. This allowed us to respond to different integration constraints with confidence.
