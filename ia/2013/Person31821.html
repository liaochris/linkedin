David is available for work to start immediately. Please contact him if you are looking for an experienced SAS/WPS Consultant specializing in generating analysis datasets and TLFs needed for a clinical study.

David has been a SAS Programmer for since 1985, and in the Pharmaceutical sector since 1991. He has mainly worked in the Biostatistics area creating SAS programs to generate the tables, listings and graphs for a CRT, but has done other work including raw eCRF data to SDTM conversion and ad-hoc requests for analysis from statisticians. David also runs a web page that gives tips and other information that is useful to the SAS Programmer (TheProgrammersCabin.com), and has had published and presented a number of papers at international, national, regional and local SAS related conferences since 1997, very recent presentations being:

- PharmaSUG 2013 entitled “Four Useful VBA Utilities for SAS® Programmers”, “Oh Quartile, Where Art Thou?” and “SQL, HASH Tables, FORMAT and KEY= — More Than One Way to Merge Two Datasets”
- NESUG 2012 entitled "Avoiding the 'Ooh Nasty', "Plotting Earthquake Events Using SAS®/GMAP and Data From the USGS", "Rediscovering the DATA _NULL_ for Creating a Report, And Putting That Text File Into RTF Format In a Single Datastep" and "Adding Control to Your Data Using SAS System and Dataset Options"
- WUSS 2012 entitled "Countdown of the Top 10 Ways to Merge Data", "Rediscovering the DATA _NULL_ for Creating a Report, And Putting That Text File Into RTF Format In a Single Datastep" and "PROC PRINT - the Granddaddy of all Procedures, Enhanced and Still Going Strong!"
- and many others

David has also served in roles as a Section Chair and at the Code Clinic as a Code Doctor at these conferences. See an expanded list of presentations at his web page.

The EXPERIENCE listed below is just a high level summary -- a more detailed resume can be supplied if requested.

Specialties

SAS and WPS programmer and consultant, with occasional use of R. Main work includes the production of tables, listings and graphs for a CRT. Creation of ODS RTF and PROC REPORT to create production ready output, and also used the DATA _NULL_ to create the same where needed. Conversion of raw eCRF data to SDTM. He also has experience creating Text, PDF, JPG and the occasional HTML output where required.
