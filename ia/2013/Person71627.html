Jimmy Zelinskie's Projects

  • Chihaya

    • 2013 to Present
    Team Members: Jimmy Zelinskie, Justin Li

    Chihaya is a high-performance, concurrent BitTorrent tracker written in Go. It exposes a generic interface that can be implemented for any data store and maintains the scaling properties of the given data store.

  • Present (featured on TechCrunch)

    • 2013 to Present
    Team Members: Jimmy Zelinskie, Dan Scanlon, Drew Oros

    Built in one day at HackPSU, Present is a geolocation-aware video streaming service where people can sign up with Twitter and stream from their iPhone or laptop. Streams are routed through EC2 and our Django-powered webapp is hosted on Heroku. The frontpage can determine what places are popular, and can fill itself accordingly so that users can get multiple views of one popular event.

  • Whirlpool Hashing in Go

    Team Members: Jimmy Zelinskie

    A personal project that I used in order to improve my understanding of cryptography. I implemented the whirlpool hashing algorithm in the programming language Go. It is an open source project licensed with a BSD license. It was offered to the Go development team, if they ever plan to expand their cryptography libraries.

  • Royale

    • 2013 to Present
    Team Members: Jimmy Zelinskie

    IST397D is an open application studio where students can make what they want. I ended up working on an Android game based on chess that that levels the playing field between veterans and newbies. Captured pieces can be challenged and players then have the ability to fight in another game to decide the owner of the square. Any game can be plugged in via Android Intents.

  • Turing Machine Emulator in C++

    • 2010 to 2010
    Team Members: Jimmy Zelinskie,

    The final project in PSU's CMPSC122 course is to implement a Turing Machine in C++ which parsed a simplistic language that fed instructions to the Turing Machine. The instructions inputted could then be stepped through and the state could be evaluated by the user running the application. The application also had the ability to compute the instructions as fast as possible and return the result.

  • Concurrent, Networked Blackjack in Java

    • 2011 to 2011
    Team Members: Jimmy Zelinskie

    The fifth project in PSU's CMPSC221 course is to create a client-server GUI application in Java. Most students implement a chat application, however Trevor Chipley and I decided to go above and beyond and implement a Blackjack game complete with graphics and an original protocol. The project used the standard Java libraries one would expect (Swing, etc...).

  • tRx

    • 2012 to Present
    Team Members: Jimmy Zelinskie, Jigesh Parekh

    The final project in PSU's IST311 course is to create a project that demonstrates the MVC pattern using an object oriented language like Java. Our group prototyped a healthcare application which queried a database to keep an update to list of drugs, their recall statuses, what drugs with which they conflict and maintained a personal database for the drugs currently taken by the user.
