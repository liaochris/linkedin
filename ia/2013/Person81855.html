I have adapted, re-designed and re-engineered Cognitive Match's existing AI-based learning machine to delivery of online ads. The current architecture, based on Spring 3, and a mix of technologies and frameworks like RMI, JMS, EHCache, CouchBase/membase is in essence implemented using just CORE Java components -- including low level NIO, Java6 concurrency classes, futures etc. The platform also offers customized scripting of solution delivery for complex targeting via Groovy.

The solution was designed with the cloud in mind and runs currently in the Amazon cloud via RightScale with a few colos delivered in Europe, US (East and West) and APAC to follow.

Existing business knowledge in internet and advertising has allowed me to design a scalable and flexible solution which allows us to plug in things like 3rd party data providers (Yahoo!, eXelate, Blue Kai etc) very easily and use this in the machine learning.

As with all startups I have been involved in designing and setting up the advertising infrastructure from the grounds up – and as such my responsibilities have spanned across not just designing and implementing the solution and managing and assisting developers in this process, but also getting involved in sysadmin tasks against our production servers, code reviews, recruitment and road-mapping the product.

  • AWS S3 Version-enabled buckets management

    • February 2013 to Present
    Team Members: Liviu Tudor

    This is a Java-based command line utility which provides functionality around S3 buckets with versioning enabled -- something which normal S3 browsers as of yet don't support.
    If you host your data in the Amazon cloud (AWS) and you use versioning on your S3 bucket(s) you would have noticed the lack of support (currently) from Amazon on this. This tool aims to provide some help with that.
    Project is entirely Java based and makes usage of the Amazon's AWS SDK for Java. It is intended as a command-line tool (so it can be easily integrated in things like cron jobs etc) however there are plans to to implement a GUI interface too.
    I have build this project from the grounds-up myself, pushed it to S3, provided site/documents/wiki/tickets/issue management too. Following that I have started building traction and interest in this through various social media channels and recruited a team of (at the moment) 4 other developers who are working alongside me on this. I am constantly providing roadmap guidance and getting involved in ticket management and release planning, mentoring the development team throughout the process of shaping this tool into a "proper" production-ready tool. (The whole experience gained into this project is now being compiled together by myself into a book which will hit Amazon's Kindle store soon -- I will post a link here when that is ready.)
    Of course, as it's always the case with open source projects, I am constantly looking for other enthusiasts who want to get involved in this.

  • Me Loves Videos

    • February 2013 to Present
    Team Members: Liviu Tudor

    Simple online utility written in PHP (LAMP stack) which uses the Amazon Elastic Transcoder API to convert videos in between a range of formats.
