Project Leader (Contract)

Citymax Integrated Information Systems Ltd

December 1995 – April 1997 (1 year 5 months) London

Citymax are a market leader in the financial software field. The have developed the Arrow product which is used as an interface to the Bank Of England Crest Paperless Trading System. My role at Citymax was to design and develop numerous Host Interface programs, which would allow Citymax customers to trade using their existing back end systems via Arrow and into Crest. Due to the differing Host Systems architectures of different customers, various communications protocols had to be used. These included SNA, DEC Message Queues, shared databases and batch file input. All work was undertaken in ANSI C under HP9000, DEC ALPHA and IBM RS6000 Unix platforms. The nature of this work involved a lot of customer interfacing and on-site support to assist customers through their various Crest trialling phases. During my time at Citymax, I was also involved in the development of the core Arrow product.

Programming Supervisor/Systems Architect

Missing Link Software Ltd

June 1988 – December 1993 (5 years 7 months) London

Missing Link is a software house specialising in Human Resource packages.

I joined Missing Link as a Junior Software Engineer. After preliminary training, I was involved in customisation on the DOS Human Resource package. I also used C to develop in-house utility programs to assist the development process.

I rapidly progressed through various positions in the development team, performing several different roles up to Senior Programmer, and then Programming Supervisor. Tasks generally involved programming in C or C++, and included maintenance programming on the existing package modules, documenting existing code, enhancements to the in-house developed data management system, and setting up the departmental version control system using Intersolv's PVCS.

In 1992 Missing Link commenced a complete redevelopment of their products using Open Systems technology. The JAM client/server 4GL tool kit was selected, under Windows 3.1, using Oracle, Sybase and Gupta SQLBase as the RDBMS server. As Project Manager I controlled this redevelopment from start to end, with approximately 10 customer sites testing the new product. We ported the product onto a UNIX workstation under Motif (SCO UNIX), against Oracle v7.

Following a departmental reorganisation, I was promoted to the position of System Architect, responsible for the architecture of all Missing Link products. The role encompassed all aspects of the development life cycle from analysis, design, coding, user interface, integration & testing, and on-going maintenance. I put in place coding and documentation standards for all the development tools that were used (C, C++, JAM, Oracle, etc.), and was responsible for ensuring that all work adhered to them. Missing Link were striving to achieve BS5750, and in order to achieve this a rigorous Quality Management System had to be implemented, including stringent change control procedures.
