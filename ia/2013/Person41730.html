Eamonn O'Brien-Strain's Patents

  • Recursive data naming

    • United States Patent 7,797,359
    • Issued September 14, 2010
    Inventors: Eamonn O'Brien-Strain

    This patent addresses the problem of how to build a data-store overlay on a radically-distributed ensemble of devices. It combines aspects of DNS, distributed-file systems, REST, and key-value noSQL stores. It is designed to have no single points of failure, to be resilient to network partitions, and to have no central authority.

    The described scheme defines a namespace of data object in terms of hierarchical names such as “/a/b/c/d”. There are special data objects that have the final component “.meta”. These specify the actual mechanism (e.g.. the pattern of a peer-to-peer call or an HTTP call to a particular server) for reading and writing data-objects. For example “/a/b/.meta” can specify the access for the subtree under “/a/b/”. It can do this via a DSL that has a way of expressing pattern matching of particular names in the subtree Another .meta further down the tree, such as at “/a/b/q/r/.meta can override the higher-level .meta for names in its sub-tree.

    The system is recursive in that .meta data objects are themselves stored in this system just like any other data object, with access mechanism specified by another .meta further up the tree. It is not quite turtles all the way down, because finally there is a root “/.meta” that each device independently stores as part of its configuration.

    For the system to be efficient and fault-tolerant, this system must be heavily cached, so that in the steady-state the part of an application running on a particular device will mostly be accessing data on the same device.

  • Secure incremental updates to hierarchicaly structured information

    • United States Patent 8,275,745
    • Issued September 25, 2012
    Inventors: Eamonn O'Brien-Strain

    In this scheme, a hierarchically structured file (e.g. XML or JSON) is represented as a flat set of triples of the form “elementId parentId elementValue”. Each triple is then encrypted into a pair like so “elementId encrypt(nonce, parentId,elementValue)”.


    The envisaged system is where an untrusted system is storing a large hierarchical tree, and a a trusted system wants to make incremental updates to it. With this scheme the trusted system, which knows the cryptographical key and the nonces for each element, can construct a sequence of ADD, REMOVE, and REPLACE commands that the untrusted system can execute to make the desired change to the tree. The untrusted system will not know the element contents, nor does it even know the structure of the tree.

  • Data-dependent overlay network

    • United States Patent 8,208,477
    • Issued June 26, 2012
    Inventors: Eamonn O'Brien-Strain

    This is an extension of standard peer-to-peer overlay P2P networks that can be more resilient to network partition. In a standard distributed hash table system, data objects are placed on different devices (“peers”) around the network without regard to how those data objects will be used.

    In this scheme the data objects are combined into groups, called “data clouds”, each of which acts like its own separate P2P network. Each node on the system can be a member of multiple clouds, keeping a separate dynamically-updated peer-list for each cloud.


    Doing this increases locality and increases the probability that a data-object requested from the P2P network will be on a peer that is currently accessible.
