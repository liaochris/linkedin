Andrew Newdigate's Projects

  • Value at Risk (VaR) Market Data Warehouse

    • February 2011 to October 2011
    Team Members: Andrew Newdigate

    Developed a Risk Market Data system with VaR reporting. I was involved in the server-side development of the system, which was a Java/Spring/Oracle/Camel/CXF system exposed through REST web-services to a .NET front-end. Market data feeds loaded from multiple sources including internal systems, Bloomberg, Markit, Superderivatives amongst others. The system performed overnight batch processing of data before feeding it into a Murex risk batch for VaR calculation. Integrated ElasticSearch for full-text indexing of warehouse data.

  • Stock Take System

    • January 2011 to February 2011
    Team Members: Andrew Newdigate

    Sole developer for this project. The goal was to build a replacement for Truworths existing legacy mainframe-based stock-take system. Built as a Java web application using Vaadin / Spring / Hibernate with an Oracle database backend and a client-side component for interfacing with handheld barcode scanners over RS232.

  • Financial Single View System

    • October 2011 to Present
    Team Members: Andrew Newdigate

  • Asset Allocation System

    • 2008 to 2009
    Team Members: Andrew Newdigate

    Developer on this project to replace the existing legacy asset allocation system for Barclays Global Investors. The system comprised an ExtGWT front-end web-application interfacing with a Java / Spring / MVC backend application using GWT RPC. Used Oracle Coherence caching and an Oracle database backend.

  • Index Projection System

    • 2007 to 2008
    Team Members: Andrew Newdigate

    Developed a system for managing index vendor projection data from index vendors including MSCI, S&P, FTSE and Nikkei. The system downloads index projection data from the vendors and presents forecasted index changes to fund managers for further analysis. Included a workflow component for acknowledging and approving projections.

  • Economic Indicator Data Analysis Tool

    • August 2009 to November 2009
    Team Members: Andrew Newdigate

    Built an analysis tool for detecting outlier data within an economic indicator data warehouse of thousands of data feeds. The system included a Swing (web-start) front-end for visualising outlier detection and optimising the outlier detection algorithm for individual feeds. The primary user interface was developed in SmartGWT, with a Hibernate/Swing backend.
