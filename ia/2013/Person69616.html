One of the first engineers hired to develop Intacct’s initial product launched in June 2000. Application consists of a full suite of financial accounting services for mid-sized corporations made available through a web client and via a series of XML based network services.

Architected the application-wide development framework for Intacct’s business and data layers. The business layer consists of numerous session managers that provide the exposed API’s for entities such as customers, vendors, and journal entries. The data layer makes available database independent query access between the business layer and an RDBMS. These frameworks allow for rapid development of new entities by extending a base manager class for standard business logic and auto-generating common SQL queries based on meta-data files. The work done for this project is now being used as a major part of BinaryCloud, a popular open-source PHP application platform.

Led development for the Purchase Order, Inventory Control, and Order Entry applications which were successfully released October, 2001. The team consisted of four developers. Responsibilities included participation in functional design meetings with product management, technical design and implementation of application, and interacting with customer service to resolve customer needs.

Became lead developer for Intacct’s B2B integration infrastructure. This system allows partner applications to be integrated within our application or for solution providers to use Intacct’s accounting engine as the backend for their own software. This role includes ownership of our XML gateway, providing exposure to business logic via XML APIs, and the development of any client side components or applications for integrators. Worked on a .NET desktop version of our application that synchronizes data with our servers via the XML gateway.
