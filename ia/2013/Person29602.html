Daniel James' Projects

  • Apache Solr Search Engine Integration

    • July 2012 to March 2013

    The core of Casting Networks' casting software is the largest database of talent in its industry. Integrating Solr into the search capabilities of the casting software enabled us to further empower our users to find talent for their projects with greater ease, better accuracy, and speedier results.

    I was the project's development lead, from researching search technologies to implementing and releasing the final code. We used Apache Solr alongside the SolrNet .NET client library, and further extended SolrNet to communicate with our implementation of SolrCloud via Apache ZooKeeper.

  • Look-A-Like Project - Inaugural CNI Hackathon Fall 2011

    • October 2011 to October 2011

    Casting Networks, Inc. introduced a new internal contest (Hackathon) to develop innovative new concepts over a 48-hour period. Teams of three (a developer, a designer, and a project manager) were given the opportunity to propose a project for the Hackathon, implement it, and present it to a panel of executive judges.

    Our team's project was to implement a revolutionary new way to search for talent (actors, models, etc.) by look-alikes. As the developer, I implemented a service to identify faces in actor headshots, which were then compared to other known faces in order to determine a "similarity" ranking. The facial detection was performed using Emgu CV (a .NET wrapper for OpenCV) and was combined in an algorithm that also took actor statistics into account for look-alike comparison. This innovative new feature took the first prize in the Hackathon, winning out over seven other teams.

  • Payment Integration Gateway

    • January 2011 to June 2011

    The Payment Integration Gateway is a system to centralize all credit card payment activities through a managed system that enables payment gateway fault tolerance and enables dynamic routing of transactions through an optimal payment gateway. The system, developed in C#.NET, provides a RESTful web service API for performing credit card transactions, which in turn, executes an appropriate transaction through a configured payment gateway, such as PayPal's PayFlow Pro or Authorize.net's payment processor.

  • Language Manager

    • December 2010 to January 2011

    Developed a revolutionary new website translation tool called Language Manager for ASP.NET web application. Designed to enable non-technical translators to translate a web application while only requiring very minimal development effort to implement Language Manager.

  • NetIQ Data Warehouse

    • 2007 to 2007

    This data warehouse was designed to collect server performance statistics from NetIQ, which had been employed to collect data from all of the servers in the corporation. That server performance data represented dozens of data points–CPU, memory, hard disk utilization–collected on a 5-minute interval from each of the 12,000 production servers and stored in an intermediate NetIQ data store.

    A previous attempt to wrangle this data had left the data warehouse with week-long refresh cycles and an inability to capture all of the available data from the NetIQ data stores. Through refactoring, we were able to take the week-long refresh cycle down to about 30 minutes, all while capturing all of the available data. In addition to refactoring the collection mechanisms, we also began doing analysis to identify trends and predict future usage.

    The NetIQ data warehouse was a 4 TB database with over 1 billion rows in the core set of fact tables, running on SQL Server 2005.

  • BMC Remedy ARSystem Data Warehouse

    This data warehouse was a rewrite of an old warehouse for ARSystem data that was maintained by hand-coded ETL jobs and column additions to the warehouse. The new warehouse was designed over 3 weeks to be a smart warehouse that detected changes in ARSystem and captured new information.
