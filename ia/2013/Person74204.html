Ken Farmer's Summary

Experience covers a wide range of applications, processes, industries and roles within IT with a primary focus on data architecture and programming on data warehouses, 'big data' and other analytical systems.

Chief strength is in mapping business needs to opportunities presented by data integration and data analysis tools; gathering support; and then rapidly and incrementally delivering these solutions.

Specialties include:
- Data warehousing, operational data stores, ETL
- Metadata-driven and adaptable databases
- Database modeling and development
- Database administration
- Data analysis
- Data quality management
- Information Security - IDS, FW, Vulnerability Scanning, Anti-Virus, Identity Management
- Relational Databases - DB2, Oracle, Informix, SQL Server, Sybase, Postgresql, MySQL, SQLite
- Programming - Python, shell scripts, SQL, stored procedures, C
- Requirements Gathering

Sr. Data Architect

IBM Internet Security Systems

Public Company; 10,001+ employees; IBM; Computer & Network Security industry

2003 – August 2013 (10 years) Colorado

Focus has been on the design, development, management and evangelism of IBM's Security and Compliance Data Warehouse. This platform consists of a parallel DB2 database on linux, custom Python ETL, custom scoring system, and multiple, primarily Cognos, dashboards. The first release took seventeen days. Every year since then has involved the addition of new feeds, new customers, new dashboards and new applications that emerge from the integrated data.

On the side I've developed a number of tools to make managing data easier - including a unit test framework for data warehouses, a Python toolkit for large-scale data management, a scoring framework, a declarative data validation library. I've designed others - including a data encryption library for DB2 and a password vault.

Also along the way I've often partnered with research, built an amazing threat scoring tool, consulted on a wide range of data warehouse, big data and data science projects, advocated for data architecture best practices across IBM, and teamed up with sales and marketing.

Ken Farmer's Publications

  • IT Security: A Bottom-Up Approach to ERM

    • Cutter IT Journal
    • July 2012
    Authors: Ken Farmer, John Markott, Mike Rowling, Michael Hughes

    Traditionally, ERM, and especially IT security risk management, has been implemented using a top-down, management-directed approach. The authors argue that a bottom-up risk management approach, grounded in data analytics, can not only provide a faster, more effective, and more efficient approach to enterprise-wide security risk management, but also show how ERM in general can be significantly improved.
