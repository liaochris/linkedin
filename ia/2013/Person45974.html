Frédéric Harper's Projects

  • MetroPress

    • November 2012 to Present
    Team Members: Frédéric Harper

    This Open Source project contributes to the WordPress community by allowing developers, and designers to create an interactive Windows 8 Store application using content from their self-hosted WordPress site.

    In a nutshell, this free framework is a flexible app template that can be easily customized, and extend it to create more awesome Windows 8 Store applications.
    The project has been initiated by IdeaNotion, but the code has been released on CodePlex for the community. Since that time, Frédéric is one of the contributors to the project, and helped many developers to get their own applications using this template, as creating some apps for his blogs too.

  • Radio Highway

    • August 2010 to November 2010
    Team Members: Frédéric Harper

    Frédéric created an iPhone application for an independent Internet radio named Radio Highway. The owner wanted a way to reach his audience with an application on the most popular platform at that time. Another goal of the application was also to find new listeners that would find his application, so at the same time his radio, on the App Store. As the application was also a companion to the site, it contained more information for truckers, but also for road lovers. The iPhone Radio Highway application was mainly around five specific elements: an integrated way to listen to the radio directly from the application, a daily splash screen controlled by the owner, a news section, an events section related to trucks, a media section with videos, and a screen to see the latest tweets coming from their Twitter account.

  • MobileTech

    • June 2007 to April 2010
    Team Members: Frédéric Harper

    When Frédéric joined IBI Solutions Inc, the first version of MobileTech was already built. It was a tool to help telecommunication companies to manage their materials, and their workload, but was very specific to one company. Frédéric was given the task to improve actual features, add more, and add a layer of abstraction to the solution to make it worth for more than one industry or type of work.

    The first task was to add a module to automate, as much as it was possible, their employee work order system. He had to do application integration with an external system from the client of the first customer of the solution. The second part of the project was to create a system to manipulate the data, and based on a logic created with the employee experience, was able to make it work for a bigger amount of work order in a fraction of the time it took before.

    Last but not least, the abstraction part of the application was made to take the software for a specific customer, and make it software as a service. The abstraction level was around the data, but also the customization of the different settings, and part of the Web application. This project was made using ASP.NET C#, MS SQL Server, SSIS, and SSRS.
