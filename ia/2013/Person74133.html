Wright Express

Public Company; 1001-5000 employees; WXS; Financial Services industry

2005 – 2007 (2 years) South Portland, Maine

Senior Programmer analyst working on the transaction processing team, which was responsible for processing millions of fuel card transactions a day.

Developed and maintained components of Wright Express's e.Best project: File Handling, Arbitration, Transaction Processing, and Core Processing. The applications are written in Java and run on Linux blades. Much of the control work is done with Unix Shell scripts and Control M.

Arbitration parses the sales files and decides which platform (Java or Mainframe) to process the transactions on.

File Handling is responsible for parsing a variety of sales files from different sources and loading them into an Oracle database.

Transaction Processing is a multi-threaded application that processes the data loaded by File Handling. It calculates taxes, discounts, fees as well as resolving external information to match with internal entities.

Core Processing is the end-of-day process that closes each business day for WEX. CP is a set of jobs that is run under Control-M that roll up daily totals, update internal accumulators for discounts, and feeds the data loaded by File Handling and processed by Transaction Processing to the WEX Data Warehouse and to PeopleSoft.

The Authorization server does all of the credit card authorizations for Wright Express, over 2.5 Million a day. I lead the team that added Card Security Code (CSC) functionality to the Authorizer, as well as being the lead developer for the project.

Provided 24/7 on call production support for the authorization and transaction processing servers.
