Daniel Egeberg's Projects

  • On the limitations of capturing complexity classes with programming languages

    • June 2012 to Present
    Team Members: Daniel Egeberg

    We show that if some formal programming language captures a complexity class, it will be impossible implementing all algorithms for problems in this complexity class although the language is able to compute all problems in the class.

    This is done by establishing the fact the language of some grammar is Σ^0_1 in the arithmetical hierarchy, and deciding a program’s membership of some complexity class is Σ^0^1-complete.

    Furthermore, it is shown that the above result remains true even for algorithms that are proven to be in P. This should generalize to other complexity classes, but is not done in this project.
