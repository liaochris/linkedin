Kaspar Lundsby's Summary

I have worked with development of very different kinds of systems from ERP systems over communication systems to command and control systems for military use. This has led me to a very serious approach to software development, which I see supported by agile software craftsmanship, focussing on high code quality supported by unit tests.

Over the past couple of years I have also taken on the role of scrum master in the teams I have worked in and obtained the title of Certified Scrum Master.

Specialties

Object oriented development (C#/Java), advanced data modelling, object-relational mapping, test driven development, pair programming, clean code, continuous integration, SCRUM, agile methods

Developer

Itai A/S

KOT Optagelsesportal: lead C# developer on a system that provides information about application status to education applicants for several large Danish university colleges. This system integrates data from several governmental systems into one database and presents it in an MVC3/Razor web page.

Application developer, MIP Data Model Expert

Saab AB

Public Company; 10,001+ employees; Defense & Space industry

January 2007 – December 2012 (6 years) Sønderborg, Denmark

DACCIS: C# developer on the command and control system for the Danish Army. My area of expertise was the mapping of domain data between this system and the NATO data models C2IEDM and JC3IEDM. This led to my participation in several MIP working group meetings in Germany where the data models were developed further. Over time my role in the DACCIS team also grew to that of lead developer.

SMART MCM: C# developer on the Mine Warfare Data Center application with special focus on data storage in the JC3IEDM.

MDV Integration: C# developer in a project that integrated a Mine Disposal Vehicle and a sonar, offering the user a set of new, intuitive user interfaces.

TactiCall: Java developer on the TactiCall communication system in the areas of TETRA dispatching and web management (built in GWT).

Over time, I have attained the role of scrum master, and I have become a strong proponent of agile development, test driven development, SCRUM and similar "modern approaches" to software development.
